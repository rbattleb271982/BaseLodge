CHANGE 1 — INVITE PAGE CLARITY

Audit:

* invite_landing.html
* trip_invite_token_landing.html
* any related invite templates

Current behavior appears to prominently display the inviter’s identity without clearly indicating that the visitor is not authenticated.

Goal:

Make it impossible for a first-time visitor to confuse the inviter’s identity with their own identity.

For unauthenticated users:

Add clear language similar to:

“You have been invited by”

[Avatar]

Solo Ntete

“You are not signed in.”

“Create an account or sign in to connect.”

The page should clearly communicate:

* who sent the invite
* that the visitor is not currently signed in
* that accepting the invite requires creating or signing into their own account

Review all invite-related templates and recommend the best UX implementation.

Implement the changes.

⸻

CHANGE 2 — AUTH RESTORATION LOGGING

Add logging whenever authentication is restored automatically.

I want visibility into:

* session cookie restores
* remember-me cookie restores

If Flask-Login provides signal hooks, use the appropriate mechanism.

Log examples:

[auth_restore]
method=remember_cookie
user_id=32
email=example@example.com
ua=iPhone

[auth_restore]
method=session_cookie
user_id=32

ua=iPhone

Include:

* user_id
* email
* auth source
* user agent
* timestamp

Do NOT log secrets, tokens, passwords, or cookie contents.

Provide exact implementation details.

Implement the logging.
