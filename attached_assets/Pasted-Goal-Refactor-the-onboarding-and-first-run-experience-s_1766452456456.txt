Goal
Refactor the onboarding and first-run experience so that:
Account creation is fast and minimal
Core identity is collected immediately after signup on a full-screen step
Follow-up questions are handled via quick modals
Users are clearly oriented on what to do next
Nothing feels heavy, confusing, or repetitive
This is a flow + UI refactor only.
Do not add new features or data models beyond what is specified.
1️⃣ SIGNUP SCREEN — ACCOUNT CREATION ONLY
Purpose
Create the account. Nothing else.
Signup screen must ONLY include:
First Name
Last Name
Email
Password
❌ REMOVE COMPLETELY from signup:
Rider Type
Skill Level
Pass
Any other identity or preference fields
Behavior
On submit, create the user account
Immediately route to Identity Setup screen

3️⃣ HOME / TRIPS SCREEN (FIRST LANDING)
User is now fully inside the app.
Identity is known
Home profile card is visible

One question cluster per modal
Fast, dismissible, non-blocking

6️⃣ STOP CONDITIONS (IMPORTANT)
After completion screen → no more onboarding UI
After second login (even if incomplete) → stop showing modals
Remaining fields are handled only via Profile page
7️⃣ WHAT NOT TO DO (STRICT)
❌ Do not combine signup + identity
❌ Do not re-add identity fields to signup
❌ Do not make follow-ups full screen
❌ Do not add new models or screens
❌ Do not block app usage after identity step
❌ Do not store step counters
8️⃣ DONE CRITERIA
Signup feels fast and lightweight
Identity feels intentional and focused
Follow-ups feel quick and optional
Users know what to do next
No “what am I supposed to do?” moments
Flow completes cleanly in under 1 minute
Do not ask follow-up questions.
Do not add checkpoints unless absolutely necessary.
Implement exactly as specified.
