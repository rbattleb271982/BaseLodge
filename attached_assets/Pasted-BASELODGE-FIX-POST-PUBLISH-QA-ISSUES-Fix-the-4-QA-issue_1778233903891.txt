1. BLOCK PAST-DATE EDITS FROM TRIP DETAIL DATE SHEET
━━━━━━━━━━━━━━━━━━━━

Issue:
The add-trip calendar blocks past dates, but the trip detail date-edit bottom sheet and `/api/trip/<id>/update-dates` endpoint do not.

Fix:
- Add server-side validation in the update-dates endpoint:
  - reject if start_date < today
  - return a clear JSON error
- Add client-side validation in the trip detail date sheet before POST:
  - reject if selected start date is before today
  - show the inline modal error
- Preserve existing checks:
  - both dates present
  - end date >= start date
  - overlap guard excluding current trip

Issue:
After changing resort from the trip detail resort bottom sheet, only the headline updates. Other references still show the old resort until reload.

Fix:
After successful resort save, update all visible resort-dependent DOM state, including:

- any `.trip-title`
- “See who’s going” subtext
- “What’s next?” mountain/resort link href

Use the response fields already returned where possible:
