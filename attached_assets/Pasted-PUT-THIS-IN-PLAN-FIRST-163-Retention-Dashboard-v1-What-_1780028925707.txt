#163 - Retention Dashboard v1

What & Why

Build a new admin page at /admin/retention focused on user return behavior, stickiness, and cohort health.

Activation tells us whether users reached value.
Retention tells us whether they came back.

For BaseLodge, retention must account for seasonality. Traditional SaaS retention metrics are useful, but they are not the entire story because skiing and trip planning naturally peak during winter.

Use existing production data only.
Do not add new analytics vendors.
Do not add new database tables.

Done looks like

- New admin page at /admin/retention
- New sidebar navigation item "Retention"
- Positioned between Activation and Network
- Extends existing admin console shell
- Matches current admin styling patterns
- Works on desktop and iPhone 15
- No changes to existing dashboard pages

--------------------------------------------------
SECTION 1 — RETENTION KPIs
--------------------------------------------------

Display KPI cards:

1. DAU
Daily Active Users

2. WAU
Weekly Active Users

3. MAU
Monthly Active Users

4. DAU / MAU
Stickiness ratio

Formula:
DAU ÷ MAU

Display as %

5. Returning Users L30

Users active this month who were also active in a prior month

Each KPI card should include:

- Current value
- Delta vs prior comparable period
- Small sparkline
- Short explanatory subtext
