I want to implement narrative continuity across BaseLodge using copy and conditional logic only. This builds on the canonical user states (A) and lifecycle signals (B). This is not a UI rebuild and not a new feature. The goal is to make the app feel intentional, calm, and confident at every stage.

Do NOT introduce new components, modals, tooltips, walkthroughs, or layouts. Use existing templates and structures. This is copy-level and conditional rendering work only.

Narrative States (Derived — Do Not Store New Fields):

State 1: Early Onboarding
Conditions:
	•	is_core_profile_complete == false
	•	login_count <= 1

Narrative intent:
	•	Welcoming
	•	Light guidance
	•	No pressure

Use language like:
	•	“Finish setting up your profile”
	•	“This takes about a minute”
Avoid urgency or task lists.

State 2: Profile Complete, Not Planning
Conditions:
	•	is_core_profile_complete == true
	•	has_started_planning == false
	•	login_count >= 2

Narrative intent:
	•	Confident
	•	Invitational
	•	Forward-looking

Use language like:
	•	“You’re all set”
	•	“Now put it to use”
Primary CTA should guide toward creating or joining a trip.
Do not repeat onboarding language.

State 3: Planning Started, Not Yet Fully Active
Conditions:
	•	has_started_planning == true
	•	is_active_user == false (edge cases)

Narrative intent:
	•	Affirming
	•	Directional without instruction overload

Use language like:
	•	“You’re planning”
	•	“Here’s what’s next”
No beginner or setup language.

State 4: Active User
Conditions:
	•	is_active_user == true

Narrative intent:
	•	Calm
	•	Non-instructional
	•	Empowering

Use language like:
	•	“Manage your trips”
	•	“Explore upcoming plans”
No onboarding or planning nudges should appear.

Implementation Rules:
	•	Narrative state is inferred dynamically from existing computed helpers and lifecycle signals
	•	Each major screen (Home, Profile, Trips, Friends) should:
	•	Have at most ONE guiding headline or subhead
	•	Reflect the user’s current narrative state
	•	Buttons and CTAs should change labels based on state (e.g., “Finish setup” vs “Edit profile”)
	•	Do not introduce duplicate or conflicting CTAs across states

Explicit Non-Goals:
	•	Do not add new lifecycle fields
	•	Do not add walkthroughs or tutorials
	•	Do not redesign layouts
	•	Do not introduce animations or timing-based UX

Deliverables:
	•	Conditional copy updates across key screens reflecting narrative state
	•	Removal of conflicting or repetitive messaging
	•	Short summary describing how narrative continuity is determined and where it is applied
