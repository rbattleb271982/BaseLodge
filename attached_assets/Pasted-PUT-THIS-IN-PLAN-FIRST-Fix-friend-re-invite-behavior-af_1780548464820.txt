Fix friend re-invite behavior after two users unfriend each other.

Current issue:
If User A and User B were previously friends, then one or both users delete/remove the friendship, User A should be able to send User B a new invite link and User B should be able to accept it. Right now, the invite flow can show: "This invite link is no longer valid."

Expected behavior:
- Removing a friend should only remove the active friendship.
- It should not permanently block the two users from becoming friends again.
- A new invite after removal should generate a fresh valid invite token.
- The recipient should be able to accept the new invite and recreate the friendship.
- Old consumed/expired invite links can remain invalid, but newly generated links must work.

Please investigate and then implement the safest fix.
