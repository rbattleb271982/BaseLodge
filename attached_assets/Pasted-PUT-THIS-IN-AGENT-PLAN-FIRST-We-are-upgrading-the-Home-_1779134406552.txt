==================================================
GOAL
==================================================

The Home Happening section currently shows labels like:

- Updated today
- Recently updated

But these are powered entirely by SkiTrip.created_at.

SkiTrip has no updated_at column, so labels imply edits that never happened.

We want Happening to reflect actual activity:

- trip created
- trip edited
- meaningful trip changes

without introducing a full activity system.

==================================================
PHASE 1 — updated_at SUPPORT
==================================================

Add:

updated_at = db.Column(db.DateTime, nullable=True)

to SkiTrip.

Add a safe startup migration following the existing migration pattern:

ALTER TABLE ski_trip
ADD COLUMN IF NOT EXISTS updated_at TIMESTAMP

Do NOT backfill old rows.

Null updated_at correctly means:

"never edited since this feature launched"

==================================================
STAMP updated_at ONLY ON MEANINGFUL USER ACTIVITY
==================================================

Home should reflect visible user behavior.

Update updated_at only when users make meaningful trip changes.

INCLUDE:

- trip dates changed
- resort changed
- trip_status changed
- visibility changed
- major trip edits through edit forms

EXCLUDE:

- accommodation updates
- equipment overrides
- participant-only writes
- carpool
- lesson intent
- hidden/internal metadata
- background system updates

Goal:

Happening should reflect real activity, not system churn.

Audit ALL trip write routes and return route inventory before modifying code.
