#68 - OG Tag Architecture Cleanup

What & Why

The Lite-mode session that added the new OG image introduced two problems:

1. Duplicate OG tags on every logged-in page.
OG tags were injected into analytics_head.html, a shared component that is included inside base_app.html.
Every page extending base_app.html now renders duplicate OG meta tags.

2. Three standalone pages still show the old OG image.
forgot_password.html, terms_and_conditions.html, and privacy_policy.html still point at:
/static/icons/icon-512.png

Because they also include analytics_head.html, each page now has conflicting OG images.

Done looks like

- analytics_head.html no longer contains OG meta tags.
- base_app.html remains the sole source of truth for logged-in page OG tags.
- forgot_password.html uses the new OG image URL.
- terms_and_conditions.html uses the new OG image URL.
- privacy_policy.html uses the new OG image URL.
- auth.html, invite_invalid.html, and invite_expired.html remain unchanged.
- No page produces duplicate OG tags.

Out of scope

- Changing OG titles/descriptions beyond what is specified
- Twitter card tags
- Native iOS/Android files

Steps

1. Remove OG block from:
templates/components/analytics_head.html

Delete the five:
<meta property="og:...">

lines that were added there.

2. Update these standalone templates:
- templates/forgot_password.html
- templates/terms_and_conditions.html
- templates/privacy_policy.html

Replace og:image:

/static/icons/icon-512.png

with:

https://app.baselodgeapp.com/static/images/og/BaseLodge-logo-primary-1200x630.png

3. Add if missing:
<meta property="og:image:width" content="1200">
<meta property="og:image:height" content="630">

4. Do NOT change:
- page-specific og:title
- page-specific og:description

5. Verify:
- logged-in pages contain exactly one set of OG tags
- standalone pages contain exactly one set of OG tags
- no duplicates remain

Relevant files

templates/components/analytics_head.html
templates/base_app.html
templates/forgot_password.html
templates/terms_and_conditions.html
templates/privacy_policy.html

Use checkpoints sparingly.

Do not finish until fully tested in Replit and summarize every file changed.
