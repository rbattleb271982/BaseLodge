Objective
Fix multiple UX and data consistency issues across Trips, Friends, Profile, and Equipment flows.
Do not introduce new features.
Do not change typography or font sizes.
Preserve existing data and logic unless explicitly stated.
Use checkpoints sparingly and only if absolutely required.
1. My Trips — Tile Card Invite Summary
Requirement
On the My Trips page, each trip tile must display a concise invite status summary.
Display
X invited · Y accepted
Rules
Show this directly on the trip tile (no click required)
Do not show pending/declined breakdowns here
The full invitee list and statuses must remain inside the trip detail page only
This is a display-only enhancement using existing invite/guest data
2. Trip Disappearing After Invite Acceptance (BUG FIX)
Bug
Trips sometimes disappear after a friend accepts a trip invite and the user navigates around the app.
Fix Requirements
Once a user accepts a trip invite:
They must be persistently associated with the trip as a participant
The trip must appear consistently across all relevant views (My Trips, Home, etc.)
Invite acceptance must promote the user into the same visibility logic as other participants
The trip should never disappear unless explicitly deleted or left
Audit Carefully
Invite → participant promotion logic
Owner vs guest filtering
Any query relying on invite status instead of participation
Upcoming/past filters that may exclude accepted guests
This is a data/query correctness fix, not a UX tweak.
3. Terrain vs “How I Like to Ride” — Single Source of Truth
Problem
Onboarding uses “How I Like to Ride”
Profile uses “Terrain”
These are currently treated as separate or inconsistently mapped
Fix
Map onboarding and profile to one underlying user field
Onboarding selections must:
Persist to the user profile
Display correctly on the profile page
Editing terrain on the profile must update the same field used by onboarding
Remove any duplicated or shadow fields
Result
One preference, shared across onboarding and profile, always in sync.
4. “Add Equipment” / “Add Boots” Not Clickable (BUG FIX)
Bug
On the Trip page, “Add Equipment” and “Add Boots” appear interactive but do nothing.
Fix
These must be fully clickable CTAs
Clicking either must navigate to:
Profile → Equipment section
Auto-scroll to (and ideally highlight) the relevant subsection
Reuse the existing equipment editing UI
Do not create new modals or new pages
Rule
If “Add X” is shown, it must be actionable.
5. Friends Screen — Clarify Trip Invites Toggle
Problem
The “Trip Invites” toggle on the Friends screen is ambiguous due to left-aligned label and right-aligned toggle.
Fix
Replace the ambiguous label with a sentence-style control that clearly scopes the toggle to the current friend.
Required copy
Can invite me to trips        [ ON | OFF ]
Rules
The meaning must be immediately clear without inference
Text and toggle should live on the same row and feel connected
The toggle must clearly apply to this specific friend
No behavior changes — copy and layout clarity only
Constraints
Do not add new database tables unless required to fix the disappearing-trip bug
Do not change typography, font sizes, or line-height
Do not alter invite permissions or logic beyond what’s required for correctness
Reuse existing routes and UI wherever possible
