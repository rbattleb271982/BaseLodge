Goal: Replace the current invite / QR logic with a token-based system that:

- Generates a unique invite token per inviter
- Uses a personalized landing page: “<Name> has invited you to connect on BaseLodge”
- Stores inviter context in session when the receiver clicks the link or QR
- After the receiver signs up OR logs in, automatically creates a two-way Friend connection
- Works for both QR codes and shareable links
- Uses checkpoints sparingly

High-level design:

- New model: InviteToken
- New route: /r/<token>  (invite landing page)
- /invite page will show the sender’s unique link (and QR uses the same link)
- auth() login/signup route will read session["pending_inviter_id"] and create a friendship

STEP 3 — HELPER FUNCTION TO GET/CREATE TOKEN
------------------------------------------------
Still in app.py, near your routes, add this helper:

import secrets

def get_or_create_invite_token(user):
    existing = InviteToken.query.filter_by(inviter_id=user.id).first()
    if existing:
        return existing

    token = secrets.token_urlsafe(16)
    invite = InviteToken(token=token, inviter_id=user.id)
    db.session.add(invite)
    db.session.commit()
    return invite

STEP 6 — NEW LANDING ROUTE /r/<token>
------------------------------------------------
Add this new route in app.py:

@app.route("/r/<token>")
def invite_token_landing(token):
    invite = InviteToken.query.filter_by(token=token).first()

    if not invite:
        # Invalid or unknown token
        return render_template("invite_invalid.html")

    inviter = invite.inviter
    if not inviter:
        return render_template("invite_invalid.html")

    # Store inviter in session so auth flow can use it
    session["pending_inviter_id"] = inviter.id

    # If user is already logged in, connect immediately
    if current_user.is_authenticated and current_user.id != inviter.id:
        from sqlalchemy import or_
        existing = Friend.query.filter(
            or_(
                db.and_(Friend.user_id == current_user.id, Friend.friend_id == inviter.id),
                db.and_(Friend.user_id == inviter.id, Friend.friend_id == current_user.id),
            )
        ).first()
        if not existing:
            f1 = Friend(user_id=current_user.id, friend_id=inviter.id)
            f2 = Friend(user_id=inviter.id, friend_id=current_user.id)
            db.session.add_all([f1, f2])
            db.session.commit()
        # Redirect an already-logged-in user to Friends (or Home)
        return redirect(url_for("friends"))

    # Otherwise render a special landing page for signup / login
    return render_template("invite_landing.html", inviter=inviter)

Note: adjust `url_for("friends")` to whatever your Friends list route endpoint name is. If it’s `friends_page` or similar, use that instead.

- REPLACE it with logic that reads `session.get("pending_inviter_id")`
- Use that logic after both successful signup and successful login

Pseudo-code for the relevant pieces (adapt to your existing structure):

Inside the branch where a new user is created (signup success) AND inside the branch where an existing user successfully logs in, add:

    inviter_id = session.get("pending_inviter_id")

        inviter = User.query.get(inviter_id)

                db.or_(
                    db.and_(Friend.user_id == current_user.id, Friend.friend_id == inviter.id),
                    db.and_(Friend.user_id == inviter.id, Friend.friend_id == current_user.id),
                )

Make sure this logic executes AFTER `login_user(user)` has been called so `current_user` is set.
