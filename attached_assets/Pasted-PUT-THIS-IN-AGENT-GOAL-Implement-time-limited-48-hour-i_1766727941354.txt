PUT THIS IN AGENT

GOAL
Implement time-limited (48-hour) invite links that, after signup or login, deep-link users to the My Trips page with the Friends tab active and a visible confirmation that they are connected.

SCOPE
This applies to:
- New users signing up via invite
- Existing users logging in via invite
- Invite links opened multiple times within the expiration window

Do NOT implement one-time-use links.

--------------------------------
INVITE LINK BEHAVIOR
--------------------------------

1) INVITE EXPIRY
- Invite links must expire 48 hours after creation
- Use a single constant:
  INVITE_EXPIRY_HOURS = 48
- Store expires_at on the Invite model

Fields required on Invite:
- token (string, unique)
- inviter_user_id (FK)
- created_at (datetime)
- expires_at (datetime)

No usage counter. No rate limiting.

--------------------------------
2) INVITE LINK ENTRY POINT
--------------------------------

Create or update an invite route:

/invite/<token>

Logic:
- Look up Invite by token
- If invite does not exist OR expires_at < now:
    - Render invite_expired.html with calm copy
- If valid:
    - Store invite token in session:
        session["invite_token"] = token
    - Redirect to login page

Do NOT create the friend connection yet.

--------------------------------
3) POST-AUTH HANDLING (LOGIN + SIGNUP)
--------------------------------

After a successful login OR signup:

IF session["invite_token"] exists:
- Validate the invite again (expiry check)
- Create the friend connection between:
    inviter_user_id
    newly authenticated user
  (use existing friend/connection logic; auto-accept)
- Clear the session token:
    session.pop("invite_token", None)
- Redirect to:
    /my-trips?tab=friends&connected=true

ELSE:
- Default redirect remains unchanged

--------------------------------
4) MY TRIPS PAGE – TAB DEEP LINKING
--------------------------------

Update the /my-trips route to accept query params:

- tab (default: "my_trips")
- connected (boolean)

Pass both to the template.

--------------------------------
5) UI BEHAVIOR ON MY TRIPS
--------------------------------

Tabs:
- My Trips
- Friends

If tab == "friends":
- Friends tab is active on load
- Friends content is rendered immediately

If connected == true:
- Show a non-modal confirmation banner at top of Friends tab:
    "🤝 You’re now connected"
- Banner can be session-only (no persistence required)

--------------------------------
6) EXPIRED INVITE UX
--------------------------------

Create invite_expired.html with copy similar to:

Title:
"This invite has expired"

Body:
"Ask your friend to send you a new one — it only takes a second."

No technical language. No error codes.

--------------------------------
7) CONSTRAINTS
--------------------------------

- Do NOT add usage counters
- Do NOT add invite revocation UI
- Do NOT add background jobs
- Do NOT block re-use within the 48-hour window
- Do NOT change existing default login redirects except when invite is present

--------------------------------
8) ACCEPTANCE CRITERIA
--------------------------------

- Invite link works across devices and multiple attempts within 48 hours
- Expired invites never create a connection
- Invitees always land on:
    My Trips → Friends tab
- User immediately sees confirmation that connection succeeded
- Session invite state is cleared after successful auth

END
