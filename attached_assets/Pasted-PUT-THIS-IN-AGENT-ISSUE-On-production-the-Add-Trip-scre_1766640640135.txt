GOAL
Make Country, State, and Resort selects fully functional and native on:

IMPLEMENTATION STEPS
--------------------------------

1. HTML STRUCTURE FIX
In templates/add_trip.html:

Wrap EACH select in a dedicated wrapper div.

Example:
<div class="native-select-wrapper">
  <select name="country" id="country">...</select>
</div>

Do this for:
- Country
- State / Region
- Resort

--------------------------------

2. CSS FIXES (ADD TO static/css/style.css)

Add the following EXACTLY:

/* ===== Mobile Select Hardening (iOS + Android) ===== */

.native-select-wrapper {
  position: relative;
  z-index: 20;
}

select {
  appearance: auto;
  -webkit-appearance: menulist;
  background-color: #1e1e1e;
  pointer-events: auto;
  touch-action: manipulation;
}

/* Prevent clipping that breaks iOS selects */
.card,
.trip-basics-card,
.form-section {
  overflow: visible !important;
}

/* backdrop-filter breaks iOS form controls */
.card,
.trip-basics-card {
  -webkit-backdrop-filter: none !important;
  backdrop-filter: none !important;
}

/* Safety: prevent overlays from blocking taps */
.native-select-wrapper * {
  pointer-events: auto;
}

--------------------------------

3. Z-INDEX & LAYOUT SAFETY

- Ensure bottom navigation bar z-index is LOWER than 20
- Ensure no parent container of the selects has:
  - overflow: hidden
  - position: fixed
  - pointer-events: none
  - touch-action: none

--------------------------------

4. JAVASCRIPT SAFETY CHECK

- Do NOT call preventDefault() on touchstart / touchend for select containers
- Do NOT attach click handlers to select parents
- Leave selects fully native
