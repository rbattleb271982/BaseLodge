To remove any remaining ambiguity, here are the final locked UX clarifications for this batch. Please implement exactly as described.
1. Home vs Profile terminology
The Home page is the main logged-in landing page.
The Profile header / profile summary is the top section of Home (above the feeds).
Feeds (My Trips / Friends’ Trips) are not profile surfaces.
Rule:
Identity elements (Equipment, Mountains Visited) appear in the profile header on Home, not inside feeds.
2. Mountains Visited display (UPDATED)
Do not use pills.
Use an expandable list in the profile header on Home.
Behavior:
Default (collapsed): show a short inline summary
Tap expands inline to show the full list
Tap again collapses
Formatting:
Alphabetical order
Format: Mountain Name, State (e.g. Vail, CO)
Plain text list (no icons, no chips)
Editing:
Display-only on Home
Tapping the section navigates to Settings → Mountains Visited for editing
3. Date input placeholder
Use placeholder text: mm/dd/yy
Placeholder disappears on focus
Placeholder reappears if field is cleared
4. Open Dates calendar selection state
Selecting a date immediately renders it in the existing maroon selected color
Deselecting returns it to default
No beige or intermediate selection state
No new colors
5. Selected dates summary (UPDATED)
Display a live-updating summary below the calendar and above the “Save Open Dates” button.
Format:
Jan: 1, 2, 3, 4
Feb: 14, 15
Rules:
Group by month
Always show month labels
Use abbreviated month names
Sorted numerically
Updates live on select/unselect
6. Date summary interaction (NEW CLARIFICATION)
Month labels are clickable
Tapping a month label (e.g. Jan) jumps the calendar to that month
Individual date numbers are NOT clickable
Summary is navigation-only, not editing
No animations required — instant jump is fine.
7. Animations
No animations or transitions required
Instant state changes are preferred
8. Explicit non-goals
Please do not add:
Tooltips
Confirmation modals
Hover states
Swipe gestures
Analytics events
These clarifications are final and locked.
Once implemented, we should be in QA-only mode for this batch.
