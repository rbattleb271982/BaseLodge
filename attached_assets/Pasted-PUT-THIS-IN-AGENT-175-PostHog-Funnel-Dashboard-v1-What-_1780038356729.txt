#175 - PostHog Funnel Dashboard v1

What & Why

PostHog instrumentation is now live through the activation journey.

Build a dedicated admin page:

/admin/posthog-funnels

This page should visualize actual PostHog funnel progression rather than database-derived proxy metrics.

Goal:

Answer:

- Where are users dropping off?
- Which activation steps are weakest?
- How long does each step take?
- Which activation actions correlate with trip creation?

Use existing PostHog implementation only.

Do not add:

- New events
- New vendors
- New tracking systems

--------------------------------------------------
PHASE 1 — FEASIBILITY AUDIT
--------------------------------------------------

Before building:

Audit whether PostHog events are flowing and available.

Verify availability of:

- signup_started
- signup_completed
- onboarding_completed
- pass_added
- availability_added
- wishlist_added
- trip_created
- friend_connected
- invite_generated

For each event provide:

- Event count
- Earliest timestamp
- Latest timestamp
- Data confidence

If insufficient event volume exists:

Recommend delaying implementation.

Do not build if there is not enough data.
