1️⃣ Home profile card context

Answer: Yes — this refers to the user’s own profile summary card on the Home screen.
	•	The Home profile card (your own card) is a compact context
	•	It should show PRIMARY rider type only
	•	Even though it’s “your” card, it’s still a scannable, high-density surface

Rule of thumb:
	•	Home = summary
	•	Profile page = detail

So:
	•	Home card → Skier
	•	Full profile page → Skier · Snowboarder (if secondary exists)

⸻

2️⃣ Friend profile page (/friends/<id>)

Answer: This is a FULL profile view. Show Primary + Secondary.

This page is explicitly:
	•	Intentional
	•	Detail-oriented
	•	Where nuance belongs

So on /friends/<id>:
	•	If only primary → Skier
	•	If secondary exists → Skier · Snowboarder

This is the only friend-facing surface where secondary rider types are visible.

⸻

3️⃣ Implementation approach (important)

Strong recommendation: Option A

Add:
	•	display_rider_type → FULL display (Primary + Secondary)
	•	compact_rider_type_display → PRIMARY only

Why Option A is best:
	•	Explicit > clever
	•	Zero ambiguity in templates
	•	No boolean flags leaking into Jinja
	•	Safer for future contributors
	•	Easier to audit during regressions

What NOT to do:
	•	❌ Option B (context flags) — too easy to misuse
	•	❌ Option C (template-only logic) — leads to drift and inconsistency

So the model should expose:
	•	One property for compact
	•	One property for full

Templates stay dumb and safe.

⸻

4️⃣ Settings / Edit Profile display

Answer: Do NOT worry about display formatting here — the form itself is the “full” context.

Clarification:
	•	Settings is considered a full context, but
	•	You do not need to render a separate “Skier · Snowboarder” display label above the form

What matters:
	•	The radio buttons reflect primary_rider_type
	•	The checkboxes reflect secondary_rider_types
	•	Editing works correctly
	•	State persists properly

If there is any non-editable summary text shown in Settings (e.g. a header or collapsed state), then yes:
	•	That summary should use the full display format
But do not add new display text just to show it.
