Objective
Replace the static “Trip invites: Off” text in the Friends list with a clear, accessible toggle control that communicates interactivity and is easy to use on mobile.

This is a UI + interaction change only.
Do NOT change database schemas or backend logic unless strictly required to persist the toggle state.

⸻

Scope
Apply this change to:
	•	Friends list rows where “Trip invites” is shown
	•	Any other surface where trip-invite status is displayed as static text

⸻

UI Requirements (LOCKED)
	1.	Replace static text
Remove the text:
“Trip invites: Off” / “Trip invites: On”

Replace it with:
	•	Label on the left: “Trip invites”
	•	Toggle switch on the right

Do NOT show “Off” or “On” as text. The toggle itself represents the state.

⸻

	2.	Toggle behavior (critical)

	•	The entire row must be tappable
	•	Tapping the label, whitespace, or toggle all flip the state
	•	The toggle switch is a state indicator, not the only hit target
	•	Minimum tap height for the row: 44px

This is required to avoid “toggle is too small” usability issues.

⸻

	3.	Visual styling

	•	Use a standard, recognizable toggle style (iOS-like)
	•	Off state:
	•	Neutral gray toggle
	•	On state:
	•	Brand color toggle
	•	Toggle should not be visually shrunk below standard size
	•	Row should visually read as a control, not metadata

⸻

	4.	State handling

	•	Toggling updates the UI immediately
	•	State persists correctly on refresh
	•	No navigation should occur when toggling
	•	No confirmation modal required

⸻

	5.	Accessibility requirements

	•	Entire row is keyboard and screen-reader accessible
	•	Toggle has appropriate aria role (switch)
	•	State change is announced correctly to assistive tech
	•	Focus remains stable after toggling

⸻

Constraints (DO NOT DO)
	•	Do NOT use a link (“Turn on ›”)
	•	Do NOT require navigation to another screen
	•	Do NOT make only the toggle itself tappable
	•	Do NOT rely on color alone to convey state
	•	Do NOT reduce tap target size below accessibility standards

⸻

Acceptance Criteria
	•	It is visually obvious that “Trip invites” is interactive
	•	Users can toggle the setting by tapping anywhere on the row
	•	Toggle does not feel small or fiddly on mobile
	•	UI updates instantly and feels responsive
	•	No regressions in the Friends list layout

⸻

Testing Required (DO NOT SKIP)
	•	Test tapping:
	•	Toggle itself
	•	Label text
	•	Empty space in the row
	•	Test with:
	•	One friend
	•	Many friends
	•	Test on small mobile screens
	•	Verify toggle state persists correctly
	•	Verify no accidental navigation occurs

Do not finish until all tests pass.
