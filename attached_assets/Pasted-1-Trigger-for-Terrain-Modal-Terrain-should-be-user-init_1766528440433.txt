1. Trigger for Terrain Modal
Terrain should be user-initiated only, not system-pushed.
Correct approach
Terrain is accessed via Profile → Edit Profile
Do not auto-prompt it on Home
Do not show subtle nags or reminders
Optional (nice, but not required now)
A lightweight “Add favorite terrain” inline link in the Profile if unset
Explicitly do NOT
Auto-trigger terrain on first login
Block anything behind terrain
Show reminder modals
Terrain is optional personalization, not onboarding.
2. Existing Terrain Data (First Chair / Après)
Do not mutate or delete existing user data.
Correct handling
Leave legacy values (First Chair, Après) in the database
Stop showing them as selectable options going forward
When displaying terrain:
If a user has legacy values, either:
Hide them from display, or
Display only valid terrain values and ignore legacy ones
Do NOT
Backfill or auto-map legacy values
Force users to re-select terrain
Delete stored data
Forward-compatible > destructive cleanup.
3. Welcome Modal Timing
The welcome modal should appear as an overlay on Home, not as a blocking interstitial.
Correct behavior
Flow:
Signup → Identity Setup → Home loads → Welcome modal overlays Home
This preserves context and avoids feeling like another “step”
Do NOT
Insert it between Identity Setup and Home as a hard gate
Delay Home rendering
The welcome modal is orientation, not a step in a funnel.
4. Existing Progressive Terrain Modals
These should be removed entirely.
Correct action
Delete or disable the automatic progressive modals that prompt for terrain on first/second login
Do not repurpose them
Terrain should only appear via explicit user action (Profile edit)
Why
Progressive prompts contradict the new “permission-based” rule
They reintroduce the feeling of endless onboarding
5. welcome_next_steps_shown_at Flag
Reuse it if semantics still match, otherwise rename for clarity.
Preferred
Rename to something explicit:
welcome_modal_seen_at
or welcome_modal_dismissed_at
Rules
Flag should:
Be set when user taps Done
Prevent the welcome modal from ever reappearing
Do not overload it with other onboarding logic
Clarity > backward compatibility here.
TL;DR for Replit
Terrain is Profile-only, user-initiated
Leave legacy terrain data untouched, just stop surfacing invalid options
Welcome modal overlays Home after Identity Setup
Remove all progressive terrain modals
Rename or cleanly reuse the welcome modal flag
