Goal:
Add small, mobile-first 14-day trend visuals to ONLY the highest-signal Dashboard KPIs.

Do NOT turn the Dashboard into a BI tool.

Scope:
Dashboard-only.

Do NOT modify:
- Growth
- Messaging
- Trips
- User Insights
- Resorts

- chart libraries
- modals
- drilldowns
- tap-to-expand analytics
- period switchers
- filters
- health scoring
- sparkline dependencies
- new analytics systems
- PostHog
- migrations
- new models

Only add tiny inline trend visuals where operational momentum matters.

Approved KPIs for Phase 2 trends:
1. Active (30d)
2. Trips This Month

Optional:
3. New This Month
ONLY if already essentially free from existing route data.

Do NOT add trends to:
- Total Users
- Upcoming Trips
- Messaging tile
- User Insights tile
- Resorts tile
- Pass metrics
- Opt-in metrics
- Catalog metrics

Trend philosophy:
- glanceable
- subtle
- lightweight
- operational pulse
- mobile-native
- informational, not analytical

Preferred implementation:
- inline SVG sparkline
- ~14 daily data points
- no external JS/chart libraries
- no animations
- no hover interactions
- no tap interactions
- no legends
- no axes
- no gridlines

Visual requirements:
- sparkline should feel secondary to the KPI numeral
- small footprint
- neutral styling
- no loud colors
- no giant chart blocks
- no increase in dashboard clutter
- preserve the simplified executive-dashboard feel

Data requirements:
- use existing data if possible
- avoid expensive queries
- avoid complex aggregation
- 14-day daily trend only
- safe zero-state handling
- omit sparkline entirely if insufficient data exists
