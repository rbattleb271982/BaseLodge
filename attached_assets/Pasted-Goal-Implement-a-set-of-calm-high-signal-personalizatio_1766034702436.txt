Goal

Implement a set of calm, high-signal personalization improvements across Profile, Trips, and Home.
These changes must be done in dependency order, with each layer supporting the next.

This is an implementation request with sequencing required.

⸻

Important Instruction (Read First)

Before coding, please:
	1.	Identify dependencies between features
	2.	Propose a clear implementation order
	3.	Then implement features one phase at a time, validating each phase before moving on

Do NOT attempt to build everything in parallel.

⸻

Features to Implement (Final Scope)

A) Profile-level Personalization

1. Terrain Preferences (FOUNDATIONAL)
Add optional terrain preferences to profiles.

Requirements:
	•	Optional
	•	Max 2 selections
	•	Options:
	•	Groomers
	•	Trees
	•	Park
	•	Backcountry
	•	Stored on user profile
	•	Display quietly on profile pages (no icons, no badges)

This data will be used later for relevance and ordering.

⸻

B) Trip Creation Personalization

2. Smart Resort Defaults (DEPENDS ON PROFILE DATA)

When creating a trip:
	•	Default the state selector to the user’s home state
	•	Pre-sort resorts (do not filter) using this priority:
	1.	Resorts compatible with the user’s pass(es)
	2.	Other resorts in the selected state
	3.	All remaining resorts

Rules:
	•	Never hide resorts
	•	User can always override defaults
	•	No UI explanation (“smart” behavior should be invisible)

⸻

C) Home Experience Personalization

3. Countdown to Next Trip (DEPENDS ON TRIPS)

On Home:
	•	If the user has an upcoming trip, show one line at the top:
“Your next trip starts in X days”
	•	Hide entirely if no upcoming trips
	•	No urgency language
	•	No CTA required

⸻

4. Availability Match Nudge (DEPENDS ON AVAILABILITY + FRIEND DATA)

If:
	•	The user has open availability
	•	AND a friend has overlapping availability or a trip

Then:
	•	Show one lightweight nudge on Home

Example copy:
“You and 2 friends are free Jan 15–17”

Rules:
	•	Show at most one nudge at a time
	•	No notifications
	•	No urgency language
	•	Tap routes to the relevant overlaps or trip view
	•	Throttle so it doesn’t resurface repeatedly

⸻

D) Invisible Personalization (APPLIES LAST)

5. Friend Ordering by Relevance (DEPENDS ON ALL ABOVE)

Sort the Friends list using quiet relevance signals:
	•	Upcoming trip overlap
	•	Pass compatibility
	•	Shared availability
	•	(Optionally) terrain overlap

Rules:
	•	No visible scores
	•	No labels
	•	No “sorted by” text
	•	This should feel natural, not explained

⸻

E) Copy-level Personalization (NO DEPENDENCIES)

6. Rider-Aware Copy
Adjust microcopy based on rider type:
	•	“Your skis” vs “Your board”
	•	“Add ski day” vs “Add ride day”

No slang. No tone change beyond specificity.

⸻

7. Seasonal Awareness (NO DEPENDENCIES)
Adjust empty states and helper text based on time of year:
	•	Pre-season: planning language
	•	Mid-season: active language
	•	Spring: closing-season language

Only copy changes — no layout changes.

⸻

Required Implementation Order

Please implement in this order and confirm each step before proceeding:
	1.	Terrain Preferences (profile data)
	2.	Smart Resort Defaults (trip creation)
	3.	Countdown to Next Trip (home)
	4.	Availability Match Nudge (home)
	5.	Friend Ordering by Relevance (friends list)
	6.	Rider-Aware Copy
	7.	Seasonal Awareness

Do not skip steps or reorder without explaining why.
