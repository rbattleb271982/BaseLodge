We are doing a mobile UX refinement pass for BaseLodge. Focus ONLY on UI/UX improvements. Do not change backend logic or data models.

Goal: Make the app feel more premium, more readable, and more intuitive on mobile (iPhone Safari first). This is a polish pass, not a redesign.

---

### 1. TYPOGRAPHY SYSTEM IMPROVEMENTS

Update typography across the app:

- Increase base font sizes slightly across all screens (especially onboarding and primary flows)
- Headline text:
  - Increase size
  - Increase font weight
  - Improve visual hierarchy
- Body text:
  - Slightly increase size for readability
  - Keep lighter weight than headlines
- Buttons:
  - Increase font size slightly
  - Increase font weight for stronger presence

Do NOT introduce new fonts. Work within existing system.

---

### 2. ONBOARDING FLOW IMPROVEMENTS

#### Add progress clarity
- Keep existing progress bar
- Add text label above headline:
  - “Step X of Y” (e.g., Step 1 of 5)
- Ensure this updates dynamically per step

#### Improve copy clarity
Update onboarding copy:
- “How do you ride?” → “Do you ski or snowboard?”
- “Where are you based?” → “Which state do you live in?”

#### Selection state improvements
- Make selected options more visually obvious:
  - Darker red/pink fill OR stronger border contrast
  - Must feel clearly “selected” at a glance

#### Layout improvements
- Reduce vertical spacing slightly to avoid unnecessary scrolling
- Ski pass selection:
  - Reduce vertical padding OR
  - Convert to 2-column layout if needed
  - Goal: avoid “barely requires scrolling” scenario

#### CTA improvements
- Final onboarding CTA:
  - Change “Finish” → “Let’s Go!”

---

### 3. EMPTY STATE + HOME SCREEN REFINEMENT

#### Home screen (first-time user)
- Move content higher up (reduce excessive top spacing)
- Tagline:
  - Increase size
  - Center align
  - Increase weight
  - Make it visually dominant
- Reduce vertical whitespace between sections

---

### 4. CONSISTENCY ACROSS TABS

#### Trips vs Ideas mismatch
- Align visual structure between:
  - Trips tab (empty state)
  - Ideas tab (empty state)

- Make Trips tab match Ideas tab style:
  - Use same background treatment (white card style)
  - Consistent padding, spacing, and hierarchy

---

### 5. STAT BAR INTERACTIVITY (HOME SCREEN)

Make stat bar actionable:

- Trips → navigate to Trips tab
- Mountains Visited → navigate to Profile → Mountains section
- Wishlist → navigate to Profile → Wishlist section

Ensure tap targets are large and responsive.

---

### 6. INVITE FLOW (FRIENDS TAB)

Add a clear “Invite friends” action:

- Add button or link in Friends tab
- On tap:
  - Open native share sheet (iOS/Android)
  - Pre-fill message:

"Hey! I’m using BaseLodge to plan ski trips with friends. You can see where people are going and find weekends that overlap. It’s free — check it out: baselodgeapp.com"

- Include app link in message

---

### 7. BUTTON HIERARCHY CLEANUP

Standardize buttons:

- Primary: filled (main actions like Continue, Let’s Go, Plan a Trip)
- Secondary: outlined (Invite friends, etc.)
- Tertiary: text links

Ensure consistency across all screens.

---

### 8. SPACING + MOBILE RHYTHM

- Reduce unnecessary vertical whitespace across screens
- Ensure consistent spacing system:
  - Between sections
  - Between cards
  - Between headings and content

Goal: eliminate “floaty” feeling and tighten layout.

---

### CONSTRAINTS

- Do NOT use checkpoints unless explicitly instructed
- Do NOT modify backend logic
- Do NOT introduce new design systems or frameworks
- Keep all changes mobile-first

---

### SUCCESS CRITERIA

- App feels tighter, more readable, and more intentional
- No unnecessary scrolling in onboarding
- Selection states are obvious
- Empty states feel consistent across tabs
- Home screen feels structured, not sparse
- Invite flow is usable and native-feeling
