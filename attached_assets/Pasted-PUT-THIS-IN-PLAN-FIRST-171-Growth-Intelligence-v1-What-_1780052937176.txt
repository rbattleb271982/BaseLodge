#171 - Growth Intelligence v1

What & Why

Activation Intelligence answers:

"Are users becoming active?"

Growth Intelligence should answer:

"Is the BaseLodge network getting stronger?"

Goal:

Answer:

* Is the user base growing?
* Is the social graph growing?
* Are invitations working?
* Are friendships increasing?
* Is the network becoming more connected?
* Are users contributing to growth?
* Is growth becoming self-sustaining?

Use existing production data only.

Do not add:

* New analytics vendors
* New databases
* New tracking systems
* New models
* New migrations

Reuse existing User, Friend, InviteToken, TripInviteToken, SkiTrip, MessageEventLog, and PostHog-backed data where applicable.

Before implementation, perform a feasibility audit.

---

## PHASE 1 — FEASIBILITY AUDIT

Audit all growth signals.

Inspect:

1. Total Users
2. New Users
3. Friend Connections
4. Connected Users
5. Invite Generation
6. Invite Acceptance
7. Friend Request Acceptance
8. Trip Invite Acceptance
9. Avg Friends Per User
10. Network Density
11. Group Trips
12. User Referrals
13. User-to-User Growth
14. WAU
15. MAU

For each signal classify:

GREEN
Directly measurable today

YELLOW
Approximation available

RED
Not reliably measurable

Provide:

* Source
* Query approach
* Current counts
* Data quality concerns
* Recommendation

Do not build until audit is complete.
