1. User Sign-Up Flow
•	Objective: Provide secure account creation and flexible sign-in options.
o	Email/Password: Users input email and password, receive a confirmation email, and are redirected to the ski pass selection screen. Includes "Forgot Password" functionality.
o	Sign in with Apple and Google: Enable login via third-party accounts, complying with respective guidelines.
o	Birthday Capture: As part of the sign-up flow, users will be required to provide their date of birth. This step ensures compliance with age-related regulations. The birthday field will be mandatory and include the following:
	Input Validation: Ensure users enter a valid date.
	Error Handling: Display a clear message if an invalid date is entered (e.g., "Please enter a valid date of birth.").
	Privacy Compliance: Ensure the date of birth is securely stored and protected according to privacy standards.
•	Error Handling: Display clear messages for invalid credentials, missing birthday input, or third-party sign-in failures.
•	Accessibility: Ensure all form fields, buttons, and error messages are accessible and support dynamic text scaling.
•	Goal: Ensure a secure, frictionless onboarding process that captures essential user data for compliance.

2. Pass Selection
•	Objective: Enable users to personalize profiles by selecting a pass.
o	Prompt users to select a pass from a dropdown menu of preloaded options.
o	Allow skipping this step if desired.
•	Accessibility: Ensure dropdowns are keyboard-navigable with clear instructions for skipping.
•	Goal: Encourage personalization while allowing flexibility for non-pass users.

3. Friend Invitations
•	Objective: Simplify the process of inviting friends to join the app.
o	Step 1: Users copy a preloaded invitation message containing a Branch.io Deep Link.
o	Step 2: Send the message through the device’s messaging app.
•	Branch.io Deep Linking:
o	Redirect users to the appropriate app store or directly into the app post-installation.
o	Embed inviter UUID as a query parameter in the Branch.io link for tracking.
•	Accessibility: All deep links must include meaningful labels for screen readers.
•	Goal: Ensure a seamless onboarding experience for invited users.

4. Push Notifications
•	Objective: Keep users engaged and informed.
•	Default State: Notifications are opt-in with an optional pre-prompt explaining benefits.
•	MVP Notification: A general reminder to check the app for updates or activity.
•	Accessibility: Notifications must be announced properly by assistive technologies.
•	Goal: Increase engagement through timely updates.

5. Friends Screen
•	Objective: Enable users to view a list of their connected friends and see their ski pass information.
•	Features:
o	Friend List Display:
o	Show a scrollable list of connected friends.
o	Each friend’s card will display:
	Profile picture (optional, placeholder if none is uploaded).
	Name or display name.
	Selected ski pass (if any).
o	Pass Visibility:
o	Indicate the ski pass (if selected) for each friend using a small icon or label.
o	Provide a fallback message like "No pass selected" if the friend hasn’t chosen a pass.
o	Next Mountain (Note - unsure if needed for MVP but definitely would like to have):
o	Display the next mountain a friend plans to ski at and the date they plan to ski there.
o	Add Friend Prompt:
o	Display a call-to-action (CTA) for users with no friends connected:
	Example: "No friends connected yet. Invite your friends to start planning together!"
	Include a button to redirect users to the Friend Invitations flow.
o	Friend Activity Placeholder (Optional for MVP):
o	Placeholder text for future updates to show activity feed (e.g., recent resort visits or updates).
•	Error Handling:
o	Display a message: "Unable to load friends. Please check your connection and try again."
o	Include a "Retry" button.
•	Accessibility Considerations:
o	Ensure all friend cards are navigable via keyboard or screen readers.
o	Use clear, descriptive labels for buttons and actions (e.g., "View Pass Details for [Friend Name]").
o	Dynamic content updates must notify assistive technologies without breaking the user flow.
•	UI Design:
o	Leverage detailed wireframes to ensure consistency with the rest of the app’s design.
o	Use standard React Native components (e.g., FlatList for listing friends).
o	Maintain color contrast and dynamic font support for accessibility.

6. Error Handling and Incomplete Actions
•	Objective: Provide clear feedback for failed actions and encourage task completion.
•	Examples: Display appropriate error messages for failed invites or sign-in issues.
•	Accessibility: Ensure error messages are accessible and focus logically moves to error locations.
•	Goal: Minimize user frustration and ensure smooth interactions.
