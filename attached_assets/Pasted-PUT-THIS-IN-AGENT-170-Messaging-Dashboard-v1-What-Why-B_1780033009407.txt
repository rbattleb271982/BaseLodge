#170 - Messaging Dashboard v1

What & Why

Build a new admin page at:

/admin/messaging

Growth tells us if users arrive.

Product tells us what they use.

Messaging tells us what drives action.

Goal:

Answer:

- Are messages being sent?
- Which message types are being used?
- Which messages drive engagement?
- Which invitation flows are working?
- Where are users dropping off?

Use existing production data only.

Do not add:

- New database tables
- New tracking vendors
- New event systems

Before implementation, perform a feasibility audit and report findings.

--------------------------------------------------
PHASE 1 — FEASIBILITY AUDIT
--------------------------------------------------

Do not build yet.

Audit all messaging-related signals currently available.

Include at minimum:

1. Friend Requests Sent
2. Friend Requests Accepted
3. Trip Invites Sent
4. Trip Invites Accepted
5. Invite Links Generated
6. Invite Links Redeemed
7. Product Push Sent
8. Product Push Delivered
9. Product Push Opened
10. Marketing Push Sent
11. Marketing Push Delivered
12. Marketing Push Opened
13. Push Opt-Outs
14. Notification Permissions
15. Email Sends
16. Email Opens
17. Email Clicks

For every signal classify:

GREEN
Directly measurable from production data with no assumptions.

YELLOW
Measurable using a proxy or approximation.

RED
Cannot be measured reliably from current production data.

Do not classify a metric as GREEN unless the underlying data is already persisted and queryable.

Provide:

- Classification
- Source model(s)
- Query approach
- Current counts
- Data quality concerns
- Recommendation (use / caveat / exclude)
