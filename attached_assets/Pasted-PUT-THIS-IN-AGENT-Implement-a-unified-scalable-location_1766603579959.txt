PUT THIS IN AGENT
Implement a unified, scalable location selector across BaseLodge that supports alphabetical ordering, country grouping, and typeahead search.
This is a global UX upgrade and should replace existing state dropdown behavior everywhere it appears.
Scope
Apply this to all location selectors, including but not limited to:
Onboarding (Home State)
Profile edit
Trip creation / edit
Filters and modals
Requirements
1️⃣ Alphabetical ordering (mandatory)
All states and provinces must be sorted alphabetically (A → Z) by full display name.
Internal values may remain abbreviations or codes (e.g., CO, CA-BC).
No dropdown should ever reflect database insertion order.
2️⃣ Group by country (mandatory)
Options must be visually grouped by country.
At minimum, support:
United States
Canada
Structure example:
United States
  Alabama
  Alaska
  Arizona
  ...
Canada
  Alberta
  British Columbia
  Ontario
  ...
Grouping must persist even when typeahead filtering is active.
3️⃣ Typeahead / searchable select (mandatory for long lists)
Replace standard <select> elements for location fields with a typeahead-enabled select when the option list exceeds ~20 items.
Behavior:
User can type to filter options instantly
Filtering works across all countries and groups
Matching should be case-insensitive and prefix-based
Must support:
Mouse
Keyboard navigation
Touch (mobile-first)
4️⃣ Shared implementation (critical)
Do not implement this separately per template.
Use one shared component, helper, or JS utility so behavior is consistent everywhere.
Future additions (more countries, resorts) should require no UX rewrites.
5️⃣ Data integrity
Do not change existing database schemas unless absolutely required.
Persist selected values exactly as before (state codes, IDs, etc.).
This is a UX and consistency upgrade, not a data migration.
Verification checklist
All state/province selectors are alphabetized
Options are grouped by country
Typing filters options immediately
Works on mobile and desktop
No regressions in onboarding, profile, or trip flows
Do not add placeholder UI, TODOs, or experimental features.
This is a foundational UX improvement and should be implemented cleanly and defensively.
