Create a surgical implementation plan for a centralized messaging event layer.

Do NOT implement yet.

Goal:
Move BaseLodge toward a cleaner communication architecture where product routes emit standardized messaging events, and a centralized orchestration layer handles delivery decisions, logging, preferences, suppression, and provider routing.

This is a planning task only.

Use checkpoints sparingly.

────────────────────────────────────────
CURRENT CONTEXT
────────────────────────────────────────

BaseLodge already has:

* MessageEventLog
* canonical messaging constants
* push notification delivery working
* OneSignal product pushes working
* APNs direct diagnostics working
* Android FCM diagnostics working
* push opt-out support
* push token hygiene
* badge clearing
* dedupe handling
* skipped/user_opted_out behavior
* route-level notification sends

The issue:
Product routes still directly trigger notification logic.

Desired direction:
Product routes should emit standardized internal events, and a centralized messaging layer should decide:

* whether to send
* who receives it
* which channel/provider to use
* whether user preferences suppress it
* whether dedupe applies
* how MessageEventLog is written
* whether future email/push/SMS/journey orchestration should happen
