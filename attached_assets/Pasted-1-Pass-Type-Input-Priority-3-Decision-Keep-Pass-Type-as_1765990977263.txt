1️⃣ Pass Type Input (Priority 3)
Decision:
✅ Keep Pass Type as MULTI-SELECT (checkboxes)
❌ Do not convert to single-select dropdown.
Why
In real life, many users legitimately have Epic + Ikon
You already support this in data + UI
Forcing single-select would:
Break existing users
Add unnecessary migration logic
Reduce accuracy
Final spec
Checkboxes for passes:
Epic
Ikon
Indy
Other
“I don’t have a pass”:
Mutually exclusive with others
Selecting it clears all other passes
Selecting any pass unchecks “I don’t have a pass”
🚫 No dropdown here.
✅ Checkboxes are correct.
Answer to Replit:
Pass Type should remain multi-select using checkboxes. Do not restrict to single-select. Handle “I don’t have a pass” as a mutually exclusive option.

3️⃣ State Lists Consistency (Important nuance)
This is a great catch — the answer is intentionally different per context.
✅ Edit Profile → Home State
Decision:
✅ All 50 U.S. states, alphabetical
Why
This represents where the user lives
Users may live in states with no resorts
This data is social / profile-related, not trip-related
✅ Add Trip → State
Decision:
✅ ONLY states that have resorts in the database
Why
Add Trip is functional
Showing empty states creates dead ends
State here is a filter, not a profile attribute
Final rule (important to document)
Context	State List
Edit Profile → Home State	All 50 U.S. states
Add Trip → State	Only states with resorts
Answer to Replit:
Use different state lists by context. Edit Profile Home State should include all 50 U.S. states alphabetically. Add Trip State should continue to be limited to states that have resorts in the database.
