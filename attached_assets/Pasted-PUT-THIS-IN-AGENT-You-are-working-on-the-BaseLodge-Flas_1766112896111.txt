Goal
Introduce a respectful, non-blocking planning-details prompt that appears on a user’s second login (not first), confirms signup completion, and optionally guides users through adding or confirming logistics-related profile details. The flow must avoid re-asking questions the user has already completed unless confirmation is lightweight and optional. No progress gauge or percentage should be shown.

––––––––––––––––
SECOND LOGIN HOME CALLOUT
––––––––––––––––

Trigger
Show this callout only on a user’s second successful login after signup.

Conditions to show:
	•	User login_count == 2 (or equivalent second-login flag)
	•	Callout has not been dismissed
	•	Core planning fields are not fully complete

Do NOT show this callout if:
	•	All core planning fields are already complete
	•	The user dismissed the callout previously
	•	The user has already completed the planning flow

Once dismissed or completed, persist a flag so the callout never reappears automatically.

Placement
Display the callout at the top of the Home screen, above other content.

Copy (use exactly this wording)

Title:
You’re all set 🎉

Body:
You’ve completed sign up. Want to add a few planning details your friends will see, or jump right into creating a trip?

Primary CTA:
Add planning details

Secondary CTA:
Create a trip

Tertiary (text link):
Skip for now

Behavior
	•	Add planning details → launches the Planning / Logistics flow below
	•	Create a trip → routes directly to trip creation and dismisses the callout
	•	Skip for now → dismisses the callout permanently without blocking anything

COMPLETION LOGIC
––––––––––––––––

A user’s planning profile is considered complete when all core logistics fields are present:
	•	Equipment status
	•	Skill level
	•	Home mountain / region (if required in the app)

Historical pass data and other enrichment fields do NOT affect completion.

If the user:
	•	Completes missing core fields → mark planning profile complete
	•	Confirms existing values without changes → mark complete
	•	Skips the entire flow → treat as acknowledged and do not nag

Do not show a completion gauge, progress bar, or percentage.
