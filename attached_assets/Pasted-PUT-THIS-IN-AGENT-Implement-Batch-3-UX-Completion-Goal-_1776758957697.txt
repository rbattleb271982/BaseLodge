## 2.1 Trip Creation UX

Current state:
- Trip creation works
- Trip editing works
- But the trip form is long and dense
- Mobile usability can be improved significantly

Goal:
Make trip creation and editing feel simpler, more guided, and less overwhelming on mobile.

### Required improvements

#### A. Reduce perceived form length
Audit the current trip create/edit form and improve the experience so it no longer feels like one long wall of inputs.

Choose the cleanest implementation based on the current structure, such as:
- grouping fields into clear sections with stronger headers
- progressive disclosure for optional fields
- or a lightweight multi-step / staged flow if that can be done cleanly without destabilizing the app

Important:
- prefer the smallest clean improvement that materially improves mobile usability
- do not overengineer a giant wizard if the form can be fixed through grouping and disclosure

#### B. Improve field grouping and hierarchy
Make the trip form easier to scan by grouping fields into intuitive sections such as:
- Basics
- Trip Details
- Gear / Ride Setup
- Visibility / Sharing
- Optional Info

Use clear section headers and spacing.

#### C. Make optional fields feel optional
If there are advanced, less-common, or optional fields:
- visually de-emphasize them
- collapse or tuck them under an "Optional details" area if appropriate
- avoid confronting the user with every field immediately

#### D. Improve mobile interaction
Audit the form for:
- field spacing
- tap targets
- select/dropdown usability
- date input clarity
- button placement
- keyboard overlap / awkward stacking where obvious from the current templates

Make practical improvements that reduce friction.

#### E. Improve submit flow
Ensure:
- the submit CTA is clear
- the save/update button placement feels intentional
- create vs edit modes are clearly labeled
- validation messages remain understandable

#### F. Preserve current logic
Do not break:
- create flow
- edit flow
- field persistence
- backend validation
- any existing trip ownership / participant logic

## 2.2 Profile UX

Current state:
- Profile is functional
- But editing can feel dense or less polished than the rest of the app

Goal:
Make the Profile experience cleaner, easier to scan, and easier to edit on mobile.

### Required improvements

#### A. Improve page structure
Audit the current profile page and organize it into clearer sections.

Examples:
- Identity
- Riding Style
- Passes / Preferences
- Availability / Social / App settings
- Notifications / Activity

Use strong grouping and visual hierarchy.

#### B. Make editing feel cleaner
If profile editing is currently too dense:
- improve layout and spacing
- better separate editable areas
- clarify which fields are primary vs secondary

Do not build a giant settings redesign unless needed.
Prefer thoughtful refinement.

#### C. Improve scanability
A user should be able to quickly understand:
- who they are in the app
- what pass they have
- what rider type / level they set
- where key actions live
- where notifications now live

#### D. Keep the new notification entry integrated
Preserve the notifications row / activity section added in Batch 2.
Make sure it still feels well-placed after profile cleanup.

#### E. Preserve current functionality
Do not break:
- profile editing
- account settings
- notifications entry
- any existing pass / rider / preference logic
