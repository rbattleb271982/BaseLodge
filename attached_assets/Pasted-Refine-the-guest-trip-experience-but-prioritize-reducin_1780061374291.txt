Refine the guest trip experience, but prioritize reducing visual noise and preserving the premium BaseLodge aesthetic.

Do NOT add new cards, banners, info panels, alert boxes, callout blocks, or additional section containers.

Problem:
The current trip detail page is already visually busy. We need to help guests understand why owner actions are missing and give them a way to leave a trip, without adding more UI clutter.

Desired behavior:

OWNER TRIPS
- Keep Invite Friends card.
- Keep Cancel Trip functionality.
- Simplify the bottom of the page:
  - Remove the "Danger Zone" heading.
  - Remove the explanatory sentence:
    "This removes the trip for everyone going."
  - Replace with a simple text link:
    "Cancel trip"
  - Keep confirmation messaging inside the confirmation sheet/modal.
- Goal: reduce visual weight and reclaim space.

GUEST TRIPS
- Do NOT add a guest information card.
- Do NOT add a guest banner.
- Do NOT add a guest callout box.

Instead:
- Add a small, subtle line of text near the trip metadata/header area:
  "Guest trip · Only the owner can edit or cancel"
- Use existing typography styles.
- Keep it lightweight and secondary.

Leave Trip action:
- Add a simple text link at the bottom of the trip detail page:
  "Leave trip"
- Style similarly to Cancel Trip.
- Do not create a new section card.
- Do not create a large CTA.

MY TRIPS PAGE
- For trips where the current user is a guest:
  - Add a small "Guest trip" indicator.
  - Optionally expose a lightweight Leave Trip action.
- Do not add buttons that increase row height significantly.
- Maintain clean list density.

Backend:
- Guests can leave a trip.
- Leaving removes only their participant record.
- Leaving never deletes the trip.
- Owners cannot use Leave Trip on their own trips.
- Owners continue using Cancel Trip.

Design principles:
- Less UI, not more.
- No profile bubbles.
- No alert banners.
- No new cards.
- Maintain BaseLodge's editorial, premium feel.
- Prefer text and hierarchy over containers.

Do not create checkpoints unless explicitly requested.
