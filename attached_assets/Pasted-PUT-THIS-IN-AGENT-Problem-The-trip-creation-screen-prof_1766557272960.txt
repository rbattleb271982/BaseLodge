Problem:
The trip creation screen, profile equipment defaults, and “Who’s Going” card are misaligned. Users cannot clearly state whether they are bringing their own equipment or renting for a specific trip, and “TBD · TBD” is ambiguous and hides which fields are missing.

Goal:
Align profile-level equipment defaults with per-trip overrides, make trip equipment selection explicit during trip creation/editing, and ensure the “Who’s Going” card always displays clear, labeled values instead of generic “TBD”.

Constraints:
- Do NOT change database schema
- Do NOT add new tables or migrations
- Do NOT change routing or business logic outside trip creation/editing and display
- Keep UX minimal and mobile-first
- Use existing fields and patterns

Implementation Steps:

1. Trip Creation / Edit Screen – Equipment Selection
Replace the current “Equipment for this trip” dropdown behavior.

Label:
Equipment for this trip

Options:
- Bringing my own equipment
- Renting equipment

Behavior:
- If the user does not explicitly select an option, inherit from the user’s profile default equipment
- Store an explicit override only when the user selects a value
- Add helper text below the field:
  “Defaults to your profile preference, but you can change it for this trip.”

2. Resolution Logic (no schema changes)
When resolving equipment for display:
- If trip-level equipment is set, use it
- Otherwise, fall back to the user’s profile default equipment
- Always resolve to a concrete value (“Bringing own equipment” or “Renting equipment”)

3. “Who’s Going” Card – Remove Ambiguous TBD
Replace the current:
“TBD · TBD”

With labeled, explicit fields.

Display format:
Equipment: {resolved_equipment} · Ride: {ride_status}

Rules:
- Never show bare “TBD”
- If a value is missing, display:
  “Equipment: Not set”
  “Ride: Not set”
- Keep formatting consistent across organizer and guests

Examples:
- Equipment: Bringing own · Ride: Has space
- Equipment: Renting · Ride: Not set
- Equipment: Not set · Ride: Not set

4. Copy Cleanup
- Remove all generic “TBD” placeholders for trip attributes
- Always label what the missing field is
- Do not introduce icons or additional UI elements

Validation Checklist:
- Users can explicitly choose bring vs rent during trip creation/edit
- Trip equipment correctly inherits from profile when untouched
- “Who’s Going” card always reads clearly at a glance
- No schema changes or regressions introduced
- Mobile layout remains unchanged
