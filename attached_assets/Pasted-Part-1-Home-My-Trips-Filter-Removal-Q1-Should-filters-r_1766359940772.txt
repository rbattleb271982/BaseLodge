Part 1 — Home → My Trips Filter Removal
Q1. Should filters remain on other Home tabs?
Answer:
✅ Yes.
Filters should be removed only from the My Trips tab, not from the entire Home screen.
Clarification by tab:
My Trips: ❌ No filters (locked)
Friends’ Trips: ✅ Filters allowed
Trip Overlaps: ✅ Filters allowed
Open Availability: ✅ Filters allowed
Rationale:
My Trips = overview, not discovery
Other tabs = browsing / matching surfaces where filters make sense
Part 2 — Friend Profile Top Card
Q2. Next trip — which one?
Answer:
✅ Show their next upcoming trip (from their perspective).
Do NOT try to compute or surface overlap logic in the top card.
Rationale:
The top card is identity, not relationship context
Overlaps belong in dedicated sections below
Q3. Equipment expand/collapse behavior?
Answer:
✅ Yes — identical behavior to Home profile card.
Same collapsed default
Same expand interaction
Same layout
Rationale:
Profile cards should be canonical and reusable
Users should never have to relearn interactions
Q4. “Mountains visited” expand behavior?
Answer:
✅ Yes — identical to Home profile card.
Same visible limit
Same “expand” affordance
Same behavior
Rationale:
Consistency > cleverness. This is intentional reuse.
Part 3 — Friends Screen
Q5. Filter control — text or icon? Show active state?
Answer:
✅ Use a text button labeled “Filter” (not icon-only).
✅ When filters are active, show state:
Either change label to “Filter (n)”
Or visually indicate active state (subtle highlight)
Rationale:
Text is clearer than an icon here
Users should know when they’re not seeing the full list
Q6. Trip count per friend — what to count?
Answer:
✅ Count only upcoming trips (future dates).
Do NOT include past trips.
Label example:
“3 upcoming trips”
Rationale:
This is a planning signal, not a history metric
Keeps the list forward-looking and useful
Q7. Pass badges in friend rows — keep or remove?
Answer:
✅ Keep pass badges visible in the friend row.
Even though filtering moves to the filter panel, pass is still:
Core identity information
Useful for quick scanning
Low-noise compared to other attributes
Rationale:
Pass ≠ filter-only metadata
Pass is foundational to ski compatibility
✅ TL;DR FOR REPLIT
Filters removed only from Home → My Trips
Friend Profile top card shows their next trip, not overlaps
Profile card interactions (equipment, mountains) are identical everywhere
Friends screen:
Text “Filter” button
Shows active state when applied
Trip count = upcoming trips only
Pass badges stay visible
