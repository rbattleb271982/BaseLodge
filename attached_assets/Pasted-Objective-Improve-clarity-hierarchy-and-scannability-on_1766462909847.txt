Objective
Improve clarity, hierarchy, and scannability on the Trips screen by:
Making tab navigation clearly tappable using outlined pill-style segmented controls
Adding clear, instructional empty states for each tab
Fixing visual hierarchy of the “Create a trip” CTA once trips exist
Updating trip cards to visually align with the new profile card design system
Improving Trip Overlaps by clearly explaining why an overlap exists
This is a UI, copy, and presentation-logic update only.
Do NOT change routing, database schemas, or business logic.
SCOPE
Apply changes to the Trips screen and its related components:
Tab navigation
Empty states
Create a Trip CTA
Trip cards
Trip Overlaps content
PART 1: TAB NAVIGATION — SEGMENTED PILL CONTROLS (LOCKED)
Convert the tab navigation into pill-shaped buttons for both active and inactive states.
Active tab:
Background: existing brand red
Text color: white
Border: none
Font weight: semibold
Inactive tabs:
Background: transparent
Border: 1px solid #D8D3CC (or closest existing neutral)
Text color: existing inactive gray
Font weight: regular
Layout rules:
Each tab must be its own pill
Large border radius (pill shape)
Minimum height: 44px
Comfortable horizontal padding
Maintain existing order, spacing, and alignment
Interaction rules:
Entire pill is clickable
On tap/click:
Active state switches correctly
Add subtle press feedback (scale to ~0.98 or slight darken for ~100ms)
Cursor pointer on desktop
Accessibility:
role="tab"
aria-selected=true on active tab
Explicitly do NOT:
Use underline-only or text-only states
Remove pills for inactive tabs
Add icons
Change tab labels or order
Change page background
PART 2: EMPTY STATES — TAB-SPECIFIC AND EXPLANATORY
Each tab must explain its purpose when empty.
My Trips (empty):
Headline: No trips yet
Supporting text: Add trips you’re planning so friends can see where you’re skiing.
Primary CTA: Create a trip
Friends’ Trips (empty):
Headline: No friends’ trips yet
Supporting text: When friends add trips, you’ll see them here.
No primary CTA
Trip Overlaps (empty):
Headline: No overlaps found
Supporting text: When you and friends are skiing at the same place or time, it’ll show up here.
No CTA
Open Avail. (empty):
Headline: No open availability yet
Supporting text: Mark days you’re open to ski to help friends plan trips with you.
Use existing CTA only if already present (do not invent new flows)
Rules:
Friendly, instructional tone
Do not reuse identical copy across tabs
Keep copy short and calm
PART 3: CREATE A TRIP CTA — VISUAL HIERARCHY FIX
If user has ZERO trips:
“Create a trip” remains the primary CTA
Full-width
Brand red background
Prominent placement
If user has ONE OR MORE trips:
De-emphasize CTA:
Reduce height slightly
Use outline or lighter background
Do not visually overpower trip cards
CTA remains visible and accessible
Do NOT change destination or behavior
PART 4: TRIP CARDS — ALIGN WITH PROFILE CARD DESIGN
Update trip cards to visually match the new profile card system.
Trip card layout rules:
Calm, vertical stack
Clear hierarchy
No dense grids
Required content (order matters):
Dates (primary)
Resort name + state
Pass used (Epic / Ikon / etc.)
Edit action (subtle, secondary)
Rules:
Dates and resort are the dominant elements
Pass should be secondary text
Edit should not compete visually with core info
Rounded container consistent with profile cards
No icons added
No new stats added
Goal:
Trip cards should feel like the same “family” as profile cards — not a separate system.
PART 5: TRIP OVERLAPS — EXPLAIN THE WHY (CRITICAL)
Each overlap must explicitly state why it exists.
For every overlap entry, show one clear reason line:
Examples:
Same resort, overlapping dates
Same resort, same dates
Same dates, different resorts
Partial date overlap at the same resort
Rules:
Exactly one explanation line per overlap
Human-readable language
No abbreviations
No raw date math shown
Optional (if already supported):
Highlight matching date range subtly
Explicitly do NOT:
Show raw start/end comparisons
Use vague labels like “Overlap detected”
Add charts or indicators
