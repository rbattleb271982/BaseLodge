GOAL

Audit the current BaseLodge product experience and identify where lightweight “How it works” explanations would meaningfully improve clarity, confidence, and understanding.

This is NOT about adding tutorials everywhere.

The goal is to identify moments where users may naturally wonder:
- “Why am I seeing this?”
- “What does this do?”
- “How does this work?”
- “Who can see this?”
- “Why was this suggested to me?”

We want:
- lightweight confidence-building
- subtle education
- editorial guidance
- reduced confusion

We do NOT want:
- onboarding overload
- enterprise-style help systems
- FAQ clutter
- excessive modals/tooltips everywhere

Perform a holistic UX/product audit across the logged-in BaseLodge experience and identify:

1.
Which screens/features would benefit from a lightweight:
- “How it works”
- “Why am I seeing this?”
- “How suggestions work”
- “Who can see this?”
- or similar contextual explanation pattern

2.
For each recommendation:
- explain WHY the clarification is valuable
- explain what user confusion it solves
- explain whether it should be:
  - inline copy
  - modal/sheet
  - tooltip
  - expandable section
  - onboarding moment
  - or NOT added at all

3.
Prioritize recommendations:
- High value
- Medium value
- Low value / unnecessary
