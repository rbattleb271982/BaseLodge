Goal:
Audit why the BaseLodge app still feels slow or laggy between screens, especially in the mobile app experience. Do not make code changes yet. I want a comprehensive diagnosis with findings ranked by expected speed/performance impact.

Important:
- Do not implement fixes.
- Do not refactor anything yet.

Context:
BaseLodge is a Flask + Jinja + SQLAlchemy app running in Replit, wrapped with Capacitor for iOS and Android. The app has a mobile-first UI with tabs such as Home, Trips, Friends, and Mountains. The app has had recent performance work, but there are still noticeable lags between screens.

Audit Scope:

1. Route-Level Speed Audit
Review the main user-facing routes and identify which are likely slowest:
- Home
- Trips
- Trip detail
- Add trip / edit trip
- Friends
- Friends trip feed
- Mountains
- Mountain detail
- Profile
- Notifications
- Any onboarding/profile setup routes

For each route, report:
- Route path
- Files/functions involved
- Main database queries
- Potential N+1 query issues
- Heavy loops or repeated queries
- Template rendering complexity
- External service calls
- Estimated performance risk: High / Medium / Low
- Expected user-visible impact

2. Database Query Audit
Review SQLAlchemy usage across app.py, routes, services, and helpers.

Look for:
- N+1 queries
- Queries inside loops
- Missing eager loading
- Repeated lookups of the same user, trips, resorts, passes, friends, or participants
- Missing indexes
- Expensive joins
- Queries that load too much data
- Queries that should be limited or paginated
- Queries that should use counts instead of loading full rows
- Queries that run on every page but could be cached or deferred

Output a table with:
- Issue
- File/function
- Current query pattern
- Why it is slow
- Suggested fix
- Expected speed impact

3. Screen Transition / Mobile WebView Audit
Because this is wrapped in Capacitor, audit anything that may cause screen-to-screen lag in the iOS/Android WebView.

Look for:
- Large CSS files or expensive selectors
- Heavy JavaScript on every page
- JS event listeners initialized globally
- DOM queries running too often
- Layout thrashing
- Large HTML payloads
- Images or assets blocking render
- Topbar/bottom nav re-render issues
- Full page reload patterns that could be optimized
- Excessive analytics or logging on page load
- PostHog scripts or OneSignal scripts affecting perceived speed

Output:
- What may be causing lag
- Where it lives
- Whether it affects all screens or specific screens
- Suggested fix
- Expected impact

- CSS
- JavaScript
- Images
- Fonts
- Analytics scripts
- Push notification scripts

Look for:
- Files loaded globally but only needed on one page
- Large unused CSS
- Duplicate JS logic
- Blocking scripts
- Non-deferred scripts
- Heavy font loading
- Images without sizing/compression
- Cache headers or static asset caching issues

Output:
- Asset/file
- Problem
- Current load behavior
- Suggested optimization
- Expected speed impact
