--------------------------------------------------
IMPLEMENTATION ADJUSTMENTS
--------------------------------------------------

The audit discovered that BaseLodge does NOT have a traditional friend-request system.

Friend growth is driven by InviteToken share links.

Do not build friend-request KPIs or funnels that imply a request/accept workflow exists.

Build only from validated GREEN/YELLOW signals.

--------------------------------------------------
SECTION 0 — MESSAGING KPIs
--------------------------------------------------

Show:

- Invitations Sent
- Invitations Accepted
- Acceptance Rate
- Friendships Formed
- Push Reach
- Push Opt-Out Rate

Definitions:

Invitations Sent
= Invite Links Generated + Direct Trip Invites

Invitations Accepted
= Invite Links Redeemed + Accepted Trip Invites

Acceptance Rate
= Accepted / Sent

Friendships Formed
= Friend table unique friendships

Push Reach
= Users with PushDeviceToken

Push Opt-Out Rate
= push_notifications_enabled=False

Display caveat icon on Push Opt-Out Rate because the field defaults to True.
