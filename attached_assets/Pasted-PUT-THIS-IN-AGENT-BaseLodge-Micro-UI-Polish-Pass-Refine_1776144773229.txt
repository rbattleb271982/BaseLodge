PART 1 — STAT BAR LABEL REFINEMENT (HOME + PROFILE)

Issue:
Labels like "MOUNTAINS VISITED" and "WISHLIST MOUNTAINS" feel cramped and slightly too heavy.

Fix:
- Slightly reduce label font size (by ~1–2px)
- Slightly reduce letter-spacing (tracking)
- Ensure labels remain uppercase and bold, but feel lighter and more breathable
- Do NOT change the numbers — those are correct (Georgia serif, large, bold)

Goal:
- Improve readability
- Reduce visual crowding
- Maintain premium editorial feel

---

PART 2 — STAT BAR DIVIDER SOFTENING

Issue:
Vertical separators between stats feel too strong compared to the rest of the UI.

Fix:
- Reduce opacity or lighten the divider color
- OR reduce thickness slightly (e.g., 1px → 0.5px or lighter tone)
- Keep them visible, but more subtle

Do NOT remove them entirely.

---

PART 3 — MOUNTAIN SEARCH RESULT PASS LABEL

Replace generic “PASS” pill with the actual pass name.

Desired behavior:
- If a mountain is on a pass, show the pass name:
  Examples:
  - Epic
  - Ikon
  - Indy

- Style:
  - Keep pill style (rounded, subtle background)
  - Use existing blush/burgundy design language
  - Ensure it clearly reads as metadata, not a button

  - Show the primary one only (do NOT show multiple pills)

- If no pass applies:

Goal:
- Make this more informative and less ambiguous

PART 4 — “YOUR TRIP DETAILS” SPACING ADJUSTMENT

Issue:
The subtext “Visible to friends when your trip is public.” feels slightly misaligned with the section header.

Fix:

Goal:
- Make the subtext feel clearly attached to the section header
- Improve visual grouping

QR code area feels slightly “floating” after removing the pink box.
