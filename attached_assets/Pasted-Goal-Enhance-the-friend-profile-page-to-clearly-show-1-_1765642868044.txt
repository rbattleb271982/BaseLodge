For each friend trip, determine whether it overlaps with any of the current user’s trips using this logic:
Same resort

friend_trip.start_date <= my_trip.end_date
AND
friend_trip.end_date >= my_trip.start_date
Dates are inclusive.
If a trip overlap exists:
Mark that trip row as has_trip_overlap = true
Visually highlight the trip row (subtle background or border)
Add a small inline line under the trip title that reads:
Both attending
If no overlap exists, render the trip normally.
Do NOT mix availability logic into trip rows.
