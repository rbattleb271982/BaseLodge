║                 EMPTY STATE NORMALIZATION                ║
╚════════════════════════════════════════════════════════════╝

Goal:
Normalize ALL empty states across the logged-in BaseLodge experience so the app feels cohesive, premium, and intentional. Empty states should follow one consistent editorial system instead of each screen using different spacing, typography, card styles, button hierarchy, or tone.

IMPORTANT:

* Mobile-first only
* Do NOT redesign populated states
* Do NOT change backend logic
* Do NOT add new routes
* Do NOT alter navigation structure
* Do NOT use checkpoints unless absolutely necessary
* Preserve all existing functionality
* Keep current maroon / cream / charcoal design system
* Follow existing BaseLodge interaction rules:

  * One primary CTA max
  * Secondary actions are text links
  * Pills are selection-only
  * Cards only for bounded content

════════════════════════════════════════════════════════════
TARGET SCREENS
════════════════════════════════════════════════════════════

Audit and normalize empty states for:

* Home
* Trips
* Friends
* Mountains
* Profile sections with no data
* Wishlist empty states
* Invite states
* Friend activity/feed empty states
* No upcoming trips
* No overlaps/opportunities
* Any “nothing here yet” states inside logged-in app

Do NOT touch admin pages.

════════════════════════════════════════════════════════════
UNIFIED EMPTY STATE SYSTEM
════════════════════════════════════════════════════════════

Every empty state should use:

1. Consistent spacing

   * Top spacing aligned across pages
   * Tight editorial rhythm
   * Avoid giant dead vertical gaps

2. Consistent typography

   * Serif headline style consistent
   * One supporting paragraph max
   * No oversized blocks of text
   * Sentence case only

3. Consistent hierarchy
   Structure should generally be:

   Small eyebrow (optional)
   Headline
   Supporting copy
   Primary CTA
   Optional secondary text link

4. Consistent CTA treatment
   Primary:

   * Full-width maroon button
   * 52px height
   * 12px radius

   Secondary:

   * Text-only maroon link
   * Optional arrow →

5. Consistent width/layout

   * Avoid random card widths
   * Avoid misaligned centered blocks
   * Respect mobile safe-area spacing

6. Consistent tone
   Tone should feel:

   * Calm
   * Editorial
   * Premium
   * Slightly social
   * Never cheesy
   * Never startup-y

════════════════════════════════════════════════════════════
COPY DIRECTION
════════════════════════════════════════════════════════════

Use short, confident copy.

Examples of acceptable tone:

* “Your season starts here.”
* “Add your first trip.”
* “The best days on the mountain don’t just happen.”
* “See where your friends are headed.”
* “Availability creates better ideas.”

Avoid:

* Emojis
* Exclamation marks
* Long onboarding-style explanations
* Generic AI-sounding filler

════════════════════════════════════════════════════════════
HOME SCREEN RULES
════════════════════════════════════════════════════════════

If user has:

* No trips
* No friends
* No availability
* No overlaps

The screen should still feel intentional and premium.

Do NOT stack giant cards endlessly.

Prioritize:

1. Add availability
2. Add first trip
3. Invite friends

“Happening” should never feel broken if empty.
Use a graceful placeholder state instead.
