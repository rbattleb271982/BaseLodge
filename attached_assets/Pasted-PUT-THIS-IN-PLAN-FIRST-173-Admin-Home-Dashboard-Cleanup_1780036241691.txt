#173 - Admin Home Dashboard Cleanup

What & Why

The admin suite now has mature dashboard categories:

Overview
- Dashboard
- Growth
- Activation
- Retention

Intelligence
- User Intelligence
- Product
- Resort Intelligence
- Messaging

Operations
- Trips
- Mountains
- User Insights
- Message Log
- Push Diag
- Resort Audit
- Resort Operations

The admin homepage should become the command center that routes administrators into those areas.

Currently it behaves primarily as a KPI page.

Goal:

Make the Dashboard feel like the front door to the admin suite.

Do not remove existing KPIs.

Do not change metric calculations.

Do not change routes.

--------------------------------------------------
SECTION 0 — EXECUTIVE KPIs
--------------------------------------------------

Keep existing KPI cards.

No metric changes.

No query changes.

No reordering.

--------------------------------------------------
SECTION 1 — OVERVIEW HUB
--------------------------------------------------

Add a new section directly beneath KPI cards.

Display 4 destination cards:

Growth
Activation
Retention
Dashboard

Each card should contain:

- Short description
- Key KPI summary
- CTA

Examples:

Growth
"How many users are arriving?"
View Growth →

Activation
"Are users reaching value?"
View Activation →

Retention
"Are users coming back?"
View Retention →

--------------------------------------------------
SECTION 2 — INTELLIGENCE HUB
--------------------------------------------------

Display 4 destination cards:

User Intelligence
Product
Resort Intelligence
Messaging

Each card should show:

- One-line purpose statement
- Most important KPI from that page
- CTA

Examples:

Product
"What features drive activation?"

Messaging
"How users invite and respond."

--------------------------------------------------
SECTION 3 — OPERATIONS HUB
--------------------------------------------------

Display cards for:

Trips
Mountains
User Insights
Tools

Purpose:

Operational workflows and diagnostics.

Do not duplicate entire dashboards.

Provide navigation only.

--------------------------------------------------
SECTION 4 — ADMIN INSIGHT
--------------------------------------------------

Display one rotating/high-value insight.

Reuse an existing insight from:

- Growth
- Activation
- Retention
- Product
- Messaging
- User Intelligence

Choose whichever insight is currently strongest.

Examples:

"Equipment users activate 3.2× more often."

"61% of users are based in Colorado."

"40% of invite links are redeemed."

--------------------------------------------------
SECTION 5 — QUICK ACTIONS
--------------------------------------------------

Create a lightweight actions section.

Include links only.

Examples:

- View Users
- View Trips
- View Mountains
- Push Diagnostics
- Message Log
- Resort Audit

No new functionality.

Navigation only.

--------------------------------------------------
MOBILE REQUIREMENTS
--------------------------------------------------

Must work on:

- iPhone 15

Requirements:

- Cards stack cleanly
- No clipped text
- No overflow
- CTAs remain tappable

--------------------------------------------------
DESIGN REQUIREMENTS
--------------------------------------------------

Match existing admin style.

Do not redesign the admin console.

Do not introduce new colors.

Do not introduce new typography.

Use existing card patterns.

--------------------------------------------------
QA REQUIREMENTS
--------------------------------------------------

Verify:

- Existing KPIs unchanged
