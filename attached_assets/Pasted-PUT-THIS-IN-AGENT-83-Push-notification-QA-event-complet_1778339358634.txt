#83 - Push notification QA + event completion pass

What & Why

The push infrastructure (OneSignal, opt-out, badge clear) is verified production-ready. This pass audits and stabilizes all four product-triggered push events end-to-end.

Two of the four events are currently stub-only (log NOT_IMPLEMENTED, send no push):

- friend.request.created — stub only, no push fires
- trip.invite.accepted — stub only, no push fires

The other two (friend.request.accepted, trip.invite.created) are wired but need copy, dedupe, and opt-out validation.

Goal:
Make all four events fully production-ready.

Audit + stabilization pass only.
No architecture changes.
