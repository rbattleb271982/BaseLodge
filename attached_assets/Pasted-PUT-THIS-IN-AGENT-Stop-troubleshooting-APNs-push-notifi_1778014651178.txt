PUT THIS IN AGENT

Stop troubleshooting APNs push notifications one step at a time. Run a full end-to-end diagnostic across the entire push pipeline and return a single clear report.

Diagnose all of these layers in one pass:

1. iOS/TestFlight app environment
- Confirm the app is loading the latest deployed app.baselodgeapp.com code.
- Confirm the user is authenticated in the TestFlight app.
- Confirm window.__USER__ exists on logged-in pages.
- Confirm Capacitor is detected as native.
- Confirm PushNotifications plugin exists under the correct Capacitor v8 API path.

2. Frontend push registration
- Confirm the push JS file/template is included on logged-in pages.
- Confirm the push script runs after logged-in page load.
- Confirm listeners are attached before registration.
- Confirm requestPermissions() is called.
- Confirm PushNotifications.register() is called.
- Confirm registration or registrationError event fires.
- Confirm the token preview is generated safely.
- Confirm POST /api/push/register-token is attempted.
- Confirm the POST response body is logged.

3. Backend token registration
- Confirm /api/push/register-token receives the request.
- Confirm current_user is authenticated at the endpoint.
- Confirm CSRF does not block the token POST.
- Confirm the token is inserted or refreshed.
- Log safe token preview only.
- Log user_id, platform, action, created_at, updated_at.

4. Token storage and selection
- List all active iOS tokens for user_id=2 using safe token previews.
- Identify whether each token is likely sandbox or production based on APNs response.
- Confirm /admin/test-push selects the latest active token.
- Confirm stale sandbox tokens are not selected if a newer production token exists.
- Consider adding token environment metadata if safely possible.

5. APNs send
- Confirm APNS_USE_SANDBOX=false.
- Confirm host is api.push.apple.com.
- Confirm bundle ID is com.baselodge.app.
- Confirm APNs key/team/key ID are present.
- Send the test push.
- Return APNs status, apns-id, reason, and explanation.

Add one temporary debug route if needed, for example:
