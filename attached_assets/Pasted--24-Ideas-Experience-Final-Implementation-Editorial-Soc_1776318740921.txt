## 2. CARD TYPES (3 TOTAL)

### A. Featured Card — Shared Availability

Only render when strong overlap exists.

Structure:

Kicker:
AN OPENING

Headline:
A shared opening

Line below:
With {FirstName} OR With {X} others (see name logic below)

Date line:
June 16 – June 19 · {Destination}

Supporting line:
On both your lists. Passes align.

CTA:
Plan trip →

Rules:
- Do NOT use “A four-day window”
- Always use “A shared opening”
- Tone must remain observational
- This is the highest priority card

---

### B. Trip Cards — Friend Trips (Planning vs Going)

Structure:

Date eyebrow:
May 30 – June 3

Headline varies based on status:

IF planning:
{FirstName} is considering {Destination}

IF going:
Join {FirstName} in {Destination}

Supporting line (pass logic below)

CTA:
Join trip →

---

### C. Wishlist Cards — Shared Pull

Structure:

Headline:
{Destination} keeps coming up

Supporting:
Several of you have {Destination} saved.

CTA:
Suggest dates →

Rules:
- No dates shown
- Lowest priority card

---

## 3. PASS COPY SYSTEM

Replace all previous pass copy with:

IF exact match:
You both have Ikon

IF mixed:
Passes vary

Do NOT use:
- “Ikon Pass”
- “Covered on your pass”
- Long explanatory phrases

Keep it short and factual.

---

## 4. PLANNING VS GOING LOGIC

Do NOT use labels.

Only reflect status via copy:

Planning:
“{FirstName} is considering {Destination}”

Going:
“Join {FirstName} in {Destination}”

Ranking adjustment:
- Planning trips rank higher than going trips (all else equal)

## 5. NAME DISPLAY LOGIC (CRITICAL)

Ideas Tab:

IF only one friend:
With Alex

IF more than one friend:
With 2 others
With 3 others

No listing multiple names on this screen.

If duplicate names:
Use first + last initial (Alex T.)

## 8. RANKING PRIORITY

Order ideas by:

1. Shared availability (multi-person)
2. Planning trips
3. Going trips
4. Wishlist pulls

Within that:

- More people > fewer
- Closer dates > far dates
- Strong pass alignment > weak
