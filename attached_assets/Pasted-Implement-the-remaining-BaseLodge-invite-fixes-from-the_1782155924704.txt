Gap 1 — Friend invite links should not expire

Update both invite GET and POST expiry logic together.

Required behavior:

* New friend invite links should be permanent
* Old unused friend invite links should still work even if their `expires_at` is in the past
* `expires_at` should no longer control friend invite validity
* `used_at` should be the source of truth

Implementation:

* In `get_or_create_invite_token()`, set `expires_at=None` for newly created `InviteToken` rows
* In `invite_token_landing()`, do not reject friend invite tokens because `token.is_expired()` is true
* Treat any friend invite token with `used_at=None` as valid forever, even if `expires_at` is in the past
* Render `invite_expired.html` only when:

  * inviter is deleted, or
  * token is already used and visitor is not already friends with inviter
* Used token + already friends should render `invite_accepted.html`
* In `invite_token_confirm()`, do not reject friend invite tokens because `token.is_expired()` is true
* Reject confirm only when:

  * token is missing
  * inviter is deleted
  * token is already used and the current user is not already friends with inviter
* Make sure expired-but-unused tokens can both render the landing page and successfully confirm

* New users should be routed back after onboarding
