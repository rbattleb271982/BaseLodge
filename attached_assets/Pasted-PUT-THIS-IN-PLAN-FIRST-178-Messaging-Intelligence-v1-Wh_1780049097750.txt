#178 - Messaging Intelligence v1

What & Why

BaseLodge now has:

* OneSignal push notifications
* MessageEventLog
* Transactional messaging
* Invite flows
* Friend request flows
* Trip invite flows

We need visibility into messaging health, delivery volume, user reachability, and engagement.

The goal is to answer:

* How many users can we reach?
* Which messages are firing most often?
* Are notifications being delivered?
* Are users opting out?
* Which messaging flows are most active?
* Is messaging helping the network grow?

Use existing production data only.

Do not add:

* New vendors
* New databases
* New tracking systems
* New event systems

Inspect and reuse the existing MessageEventLog, OneSignal, notification, and admin dashboard infrastructure wherever possible.

Before implementation, perform a feasibility audit.
