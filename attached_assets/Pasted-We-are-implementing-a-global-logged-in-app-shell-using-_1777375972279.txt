We are implementing a global logged-in app shell using Jinja template inheritance. Follow the approved migration plan exactly.
Context:
All logged-in pages are currently standalone HTML files with duplicated <head>, <body>, and bottom nav includes. This causes inconsistent safe-area handling and spacing issues. We are replacing this with a shared base_app.html.
The full migration plan has already been defined. Execute it carefully without deviating.
1. Create base_app.html
Create templates/base_app.html with:
<!DOCTYPE html>
<html>, <head>, <body>
<meta name="viewport" content="width=device-width, initial-scale=1, viewport-fit=cover">
CSS include (static/styles.css)
favicon links
{% include 'components/analytics_head.html' %}
Structure:
{% block title %}{% endblock %}
{% block head %}{% endblock %} (for page-specific CSS/meta)
<div class="app-shell">
{% block content %}{% endblock %}
{% include 'components/bottom_nav.html' %}
{% block scripts %}{% endblock %}
2. Update global CSS (static/styles.css)
Add:
.app-shell {
padding-top: env(safe-area-inset-top, 0px);
}
Ensure:
.page-container {
padding-bottom: calc(76px + env(safe-area-inset-bottom, 0px));
}
Remove or reduce any global fallback like:
body { padding-bottom: 80px; }
3. Migration rules (STRICT)
For every migrated template:
Replace full HTML with:
{% extends "base_app.html" %}
Move page title into {% block title %}
Move page-specific CSS into {% block head %}
Move all visible UI into {% block content %}
Move scripts into {% block scripts %}
REMOVE:
<!DOCTYPE html>
<html>, <head>, <body>
{% include 'components/bottom_nav.html' %}
duplicate CSS includes
4. Remove ALL inline padding overrides
Anywhere you see:
style="padding: ..."
Remove it completely.
Use only:
<div class="page-container">
Do NOT recreate padding inline.
5. Handle special layout cases carefully
DO NOT double-stack safe-area:
These already handle bottom inset:
trip_detail.html
edit_profile.html
trip_invite_detail.html
Do NOT add extra bottom padding to parent containers.
Custom wrappers
Fix these explicitly:
.mv-wrap (mountains_visited.html)
.et-page (edit_trip.html)
Ensure they use:
padding-bottom: calc(76px + env(safe-area-inset-bottom, 0px));
Local CSS overrides
Remove .page-container overrides in:
idea_detail_availability.html
idea_detail_trip.html
idea_detail_wishlist.html
