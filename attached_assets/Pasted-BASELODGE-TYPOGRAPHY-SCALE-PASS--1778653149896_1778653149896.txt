BASELODGE TYPOGRAPHY SCALE PASS             ║
╚══════════════════════════════════════════════════════════════╝

Goal

Increase readability and make BaseLodge feel more like a polished native iOS app instead of a responsive website. This is a typography-only refinement pass — no layout redesigns, no new components, and no architecture changes.

Important Constraints

* Keep the existing editorial aesthetic
* Do NOT increase giant stat numbers
* Do NOT redesign spacing systems unless required to prevent wrapping/crowding
* Do NOT touch colors
* Do NOT introduce Tailwind or CSS variables refactors
* Use checkpoints sparingly

Typography Targets

1. Primary body text
    Increase globally from roughly:

* 15px → 17px
    or equivalent existing values

Applies to:

* feed rows
* list rows
* card body text
* mountain rows
* profile metadata
* trip subtitles
* inline descriptive text

2. Secondary/meta text
    Increase globally from roughly:

* 12–13px → 14–15px

Applies to:

* “Added this week”
* timestamps
* helper text
* sublabels
* metadata rows
* empty state support text

3. Card titles / important row titles
    Increase from roughly:

* 18–20px → 21–22px

Applies to:

* mountain names
* idea card titles
* trip titles
* major row titles

4. Bottom nav labels
    Increase:

* ~12px → 14px

Ensure:

* labels still fit
* spacing remains balanced
* no icon overlap

5. Pills / segmented controls / CTA labels
    Increase slightly:

* +1 to +2px max

Maintain:

* pill proportions
* button height balance

6. Section headers
    Keep mostly unchanged.
    “Happening” and “Ideas” already feel appropriately premium.

Required QA

Test all of the following:

* Home
* Trips
* Friends
* Mountains
* Profile
* Add Trip
* Trip Detail
* Friend cards
* Bottom nav
* Empty states
* Pills/buttons
* Native iPhone viewport
* Android viewport

Specifically verify:

* no clipped text
* no overlapping labels
* no broken flex rows
* no excessive wrapping
* bottom nav still fits safely
* cards retain premium spacing
* no giant typography regression

Expected Outcome

The app should feel:

* more native
* easier to read
* more premium
* more confident visually
* less “tiny responsive website”

Do not stop at simple font increases. Adjust line-height and spacing minimally where needed to maintain polish.
