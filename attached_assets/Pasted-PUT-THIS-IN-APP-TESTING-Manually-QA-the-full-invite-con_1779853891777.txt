QA Scenarios

SECTION 1 — Friend invite links

1. Existing user → existing app user

- Verify no duplicate friendship rows

- User A sends invite

- Complete onboarding

3. Existing user → Google OAuth login
- User A sends invite
- User B clicks invite

- Verify invite context survives
- Verify redirect does NOT go to /home prematurely
- Verify friendship created

4. Wrong-account UX
- User C logged in
- Open User A invite link
- Verify “Logged in as…” warning appears
- Verify logout link works

5. Existing friends re-clicking invite
- Already-connected users open invite link again
- Verify clean behavior
- No duplicate friendship
- No errors
- No broken state

SECTION 2 — Trip invite links

6. Existing user → existing Google user
- Open trip invite
- Sign in with Google
- Verify redirect returns to trip invite flow
- Verify trip acceptance works

7. Brand-new user → trip invite
- Open trip invite
- Create account
- Complete onboarding
- Verify redirect back to trip invite flow
- Verify trip join works

SECTION 3 — QR connect flow

8. Logged-out QR scan
- Open /connect/<id>

- Log in

- Verify connection succeeds

9. Logged-in QR scan

- Verify direct connect flow works

11. Native app direct invite flow
- Open invite link directly inside app if supported

- Verify redirect behavior
