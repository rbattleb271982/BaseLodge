#121 - Happening Feed — Consistent 3-Line Content Model
What & Why
Every Happening card currently puts name + action into a single text field ("Taylor Morgan · planning Telluride") and uses "Upcoming trip" as the recency fallback — a state label, not a time label. The result is cards that mix state and recency signals on the wrong lines, produce redundant wording ("planning…" + "Upcoming trip"), or render with no action verb ("Zach · Breckenridge" + "Upcoming trip").

The fix makes every card follow a strict 3-line model: Person → Action+Mountain → Recency. No redesign, no layout change, no dismiss or cap logic change.

New content model
Data (app.py)
Replace text with two separate keys:

person — full name only, e.g. "Zach Kissinger"

action_line — verb + mountain, derived from trip_status:

'going' → "Going to {mountain}"
'planning' (or default) → "Planning {mountain}"
any other confirmed/booked status → "Heading to {mountain}"
fallback (no status, no mountain) → "Trip to {mountain}" or "Trip upcoming"
recency_label — time only, never a state word:

age < 24 h, was updated → "Updated today"
age < 24 h, was added → "Added today"
age < 7 d, was updated → "Updated this week"
age < 7 d, was added → "Added this week"
else (old or no timestamp) → "Recently updated" ← replaces "Upcoming trip"
Remove "Trip " prefix from all labels (old: "Trip added today" → new: "Added today").

Template (_section_happening.html)
Replace the single <p class="bl-happening-text">{{ signal.text }}</p> with:

<p class="bl-happening-name">{{ signal.person }}</p>
<p class="bl-happening-action">{{ signal.action_line }}</p>

Keep <p class="bl-happening-recency">{{ signal.recency_label }}</p> unchanged.

Rename .bl-happening-text → .bl-happening-name in the template only (CSS class added in home.html).

CSS (home.html)
Rename .bl-happening-text to .bl-happening-name (same 17px 500 styles — strongest line).

Add .bl-happening-action: 15px, 400, var(--bl-color-text-muted) — secondary line between name and recency. Slightly quieter than name but not as muted as recency.

.bl-happening-recency unchanged — 14px, muted, 400.
