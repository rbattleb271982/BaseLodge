🔒 PART 1 — FINAL STRUCTURAL DECISIONS (APPROVAL)
1️⃣ Lifecycle model (SIMPLIFIED)
Use only three lifecycle stages:
new
onboarding
active
Rules:
lifecycle_stage is derived by logic, not user-controlled
Transitions:
new → onboarding: user starts profile setup
onboarding → active: user performs ANY meaningful activation action:
creates a trip
joins a trip
creates a connection
sets open dates
Do not store dormant or churned as lifecycle stages
Dormancy is derived from time since last activity
Churn = account deleted
2️⃣ Activity definition (last_active_at)
Update last_active_at ONLY on login.
Do NOT update it on:
trip creation
trip joining
connections
open dates
profile edits
This field measures presence, not actions.
3️⃣ Open dates meaning
Setting open dates counts as activation-lite
A user may become active without:
creating a trip
creating a connection
4️⃣ Trip roles
Distinguish between:
trip_created
trip_joined
These are different engagement signals
Treat them as separate events
5️⃣ Seeded users
Add is_seeded flag
Seeded users must be:
Excluded from lifecycle logic
Excluded from events
Excluded from email logic
