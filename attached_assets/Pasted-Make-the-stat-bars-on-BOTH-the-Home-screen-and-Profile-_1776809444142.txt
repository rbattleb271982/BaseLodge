Make the stat bars on BOTH the Home screen and Profile screen interactive (clickable) so users can navigate directly from stats to relevant sections.

---

### GOAL

Turn each stat item into a tappable element that:
- clearly looks clickable
- navigates to the correct destination
- improves discoverability without redesigning the layout

---

### STATS TO UPDATE

Apply this to BOTH:
- Home screen stat bar
- Profile screen stat bar

Stats include:
- Trips
- Mountains Visited
- Wishlist Mountains

---

### TAP DESTINATIONS

Map each stat to the correct location:

- Trips → navigate to Trips tab
- Mountains Visited → navigate to Profile page, scroll to Mountains Visited section
- Wishlist Mountains → navigate to Profile page, scroll to Wishlist section

---

### UI / VISUAL FEEDBACK

Make it obvious these are clickable WITHOUT redesigning the component:

- Wrap each stat block in a clickable container
- Add subtle affordances:
  - cursor: pointer (desktop)
  - slight opacity or scale change on tap
  - optional subtle underline or chevron indicator
- Ensure tap targets are large enough for mobile

DO NOT:
- add heavy buttons
- change layout structure
- clutter the design

---

### MOBILE UX

- Ensure tap area is at least ~44px height
- No overlap with other elements
- No accidental taps between stat items

---

### SCROLL BEHAVIOR (PROFILE)

When navigating to Profile sections:
- Scroll smoothly to:
  - Mountains Visited section
  - Wishlist section
- Ensure proper anchor or JS scroll logic is used

---

### CONSTRAINTS

- Do not redesign stat bar layout
- Do not change existing data logic
- Keep styling consistent with current design system

---

### TESTING

Verify:
1. Tapping Trips → opens Trips tab
2. Tapping Mountains Visited → scrolls correctly to that section
3. Tapping Wishlist → scrolls correctly to that section
4. Tap targets feel natural on mobile (no mis-taps)
5. Visual feedback confirms interactivity

Do not finish until all stat items are working cleanly across both Home and Profile.
