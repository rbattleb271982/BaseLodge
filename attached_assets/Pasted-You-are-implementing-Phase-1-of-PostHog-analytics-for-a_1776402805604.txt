# STEP 2 — Create analytics.py (server)

Create a module with:

### Initialization
- Read:
  - POSTHOG_KEY
  - POSTHOG_HOST (default to https://us.i.posthog.com if not set)
- If POSTHOG_KEY is missing → all functions become no-ops

### Functions

track(user_id, event, properties={}, set_props=None, set_once_props=None)

identify(user_id, properties={})

alias(anon_id, user_id)
- This MUST alias the anonymous browser/session identity to the newly created user ID
- This is ONLY used on signup

is_internal(email)
- Returns true if:
  - email domain matches INTERNAL_EMAIL_DOMAINS (comma-separated env var)
  - OR email is in INTERNAL_USER_EMAILS list
- Otherwise false

### Rules

- Always call track AFTER db.session.commit()
- Never fire events before a successful DB write
- Use $set and $set_once correctly in PostHog payloads

# STEP 3 — Create static/analytics.js (client)

This file should:

### Initialize PostHog
- Use CDN snippet
- Use POSTHOG_KEY and POSTHOG_HOST

### Read global variables
- window.__USER__
- window.__POSTHOG_RESET__

### Behavior

1. If window.__POSTHOG_RESET__ is true:
   - call posthog.reset() immediately

2. If window.__USER__.id exists:
   - ALWAYS call posthog.identify(user.id)
   - DO NOT skip internal users

3. Fire test event:
   posthog.capture('app_loaded')
   - Fire for ALL users (anonymous, internal, external)

### Notes

- Do NOT suppress tracking for internal users
- Filtering happens later in PostHog dashboards
- Do NOT double-initialize PostHog

# STEP 5 — Wire identity in auth routes (app.py)

### Signup flow

After:
login_user(new_user)

Do:
1. analytics.alias(anon_id, user.id)
   - anon_id should come from PostHog cookie if available
2. analytics.identify(user.id, {
   is_internal: true/false
})

### Login flow

After:
login_user(user)

Do:
analytics.identify(user.id, {
  is_internal: true/false
})

### Logout flow

Before redirect:
session['ph_reset'] = True

### Auth GET route

- Pop session['ph_reset']
- Pass:
  posthog_reset=True/False into template
