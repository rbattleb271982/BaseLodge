Goal:
Implement Step 3 of the password reset flow: allowing a user to reset their password using a valid reset token.

Context:
- The User model has:
  - password_reset_token
  - password_reset_expires_at
- The /forgot-password flow is already implemented and emails a link like:
  /reset-password?token=XYZ
- There may be a placeholder /reset-password route — replace or complete it.

Requirements:

1. Implement /reset-password route
   - Methods: GET, POST
   - Accepts token via query param (?token=...)

2. Token validation (on both GET and POST):
   - Token must exist
   - Token must match a user
   - password_reset_expires_at must be in the future
   - If invalid or expired:
     - Show a generic error page/message
     - Do NOT reveal which condition failed

3. GET /reset-password:
   - If token is valid:
     - Render a form with:
       - New password
       - Confirm password

4. POST /reset-password:
   - Validate:
     - Passwords match
     - Password meets existing password rules
   - If valid:
     - Hash and save the new password
     - Clear:
       - password_reset_token
       - password_reset_expires_at
     - Commit the change
     - Redirect to a success page or login with success message

5. Security requirements:
   - Token must be single-use
   - Token must be invalidated immediately after password reset
   - Do NOT log the token
   - Do NOT auto-log the user in

6. Templates:
   - Create minimal templates:
     - reset_password.html (form)
     - reset_password_invalid.html OR inline error handling
     - reset_password_success.html
   - Styling can be minimal and consistent with auth pages

7. Do NOT:
   - Send additional emails
   - Add analytics
   - Add rate limiting
   - Modify SendGrid logic
   - Modify the login screen yet

Verification checklist:
- Valid token → password can be changed once
- Reusing the same link fails
- Expired token fails
- Old password no longer works
- New password works
- Token fields are cleared after success

Output:
- Show the completed /reset-password route
- Show password hashing logic
- Show templates added
- List all files changed
