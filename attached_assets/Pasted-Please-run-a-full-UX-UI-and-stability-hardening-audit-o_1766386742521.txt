Please run a full UX, UI, and stability hardening audit of the app with a focus on completeness and edge cases.
Review the application as if it’s about to be used by real users who:
Have empty data
Click things out of order
Refresh pages
Navigate directly via URLs
1. Empty States (Critical)
For every screen, section, and component, confirm:
What shows when there is no data
Whether the empty state:
Clearly explains what’s missing
Tells the user what to do next
Includes a clear CTA when appropriate
Explicitly check empty states for:
No trips
No upcoming trips
No past trips
No friends / connections
No invitations
No gear / equipment selected
No pass selected
No profile fields completed
Flag any screen that:
Renders blank
Looks broken
Has dead space with no guidance
2. Buttons, CTAs & Save Actions
Audit every screen and confirm:
Save buttons exist where users can edit data
Buttons are not duplicated or conflicting
Button labels clearly match their action
Disabled states exist when required inputs are missing
There is always a clear “what now?” after clicking
Explicitly flag:
Editable screens without a Save / Confirm action
Buttons that appear but do nothing
Actions that silently fail
3. Navigation & Dead Ends
Verify that:
No screen traps the user with no next step
Back links make sense and don’t feel random
Users can always get back to Home / Trips
Deep links (refreshing, direct URL entry) don’t break
4. Error States & Guardrails
Identify and flag:
Any routes that could throw a 404 or 500
Templates that assume data exists
Missing if checks around optional fields
Forms that can submit partial or invalid data
Confirm:
Custom handling for 404 and 500 errors
User-friendly messaging instead of raw errors
5. Consistency & Polish
Check for:
Inconsistent button placement
Mismatched labels (e.g., “Save” vs “Update”)
Screens that feel unfinished compared to others
Leftover debug text, test copy, or placeholders
6. Launch Safety Checklist
Identify:
🔴 Anything that could confuse or block a real user
🟡 Anything that feels unpolished but usable
🟢 Nice-to-have improvements
Output requirements:
Organize findings by category
Be specific (file, screen, or route when possible)
Do not make changes yet — audit only
Prioritize clarity over volume
