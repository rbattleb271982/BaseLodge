Goal
Implement calm, high-signal personalization across Profile, Trips, Home, and Friends.
All work must follow dependency order and preserve an Apple/Airbnb-style UX.

This is an implementation request with sequencing required.

⸻

IMPORTANT
Before coding:
	1.	Identify dependencies
	2.	Follow the implementation order exactly
	3.	Complete each step before moving to the next

Do not parallelize dependent work.

⸻

REQUIRED IMPLEMENTATION ORDER
	1.	Terrain Preferences
	2.	Smart Resort Defaults
	3.	Countdown to Next Trip
	4.	Availability Match Nudge
	5.	Friend Ordering by Relevance
	6.	Rider-Aware Copy
	7.	Seasonal Awareness

Do not reorder without explanation.

⸻

	1.	TERRAIN PREFERENCES (PROFILE-LEVEL, FOUNDATIONAL)

Purpose
Provide a quiet compatibility signal that helps friends understand how someone likes to ski.

Data rules
	•	Store terrain preferences as a JSON-style list (array), not a comma-separated string
	•	Optional
	•	Maximum 2 selections
	•	Options:
Groomers
Trees
Park
Backcountry

Editing location
	•	Add to the existing Profile section in Settings
	•	Do not create a new standalone section

Display rules
	•	Show on the user’s own profile
	•	Show on friend profile pages
	•	Secondary text only
	•	No icons, badges, or emphasis
	•	Hide the section entirely if empty

⸻

	2.	SMART RESORT DEFAULTS (TRIP CREATION ONLY)

Purpose
Reduce friction during trip creation using invisible personalization.

Behavior
When creating a new trip:
	•	Default the state selector to the user’s home state
	•	Pre-sort resorts (do not filter) in this order:
	1.	Resorts compatible with the user’s pass or passes
	2.	Other resorts in the selected state
	3.	All remaining resorts

Rules
	•	Never hide resorts
	•	User can always override
	•	Do not apply when editing existing trips
	•	Do not explain this behavior in the UI

⸻

	3.	COUNTDOWN TO NEXT TRIP (HOME)

Purpose
Create anticipation without noise.

Behavior
If the user has any upcoming trip (host or guest):
	•	Show one line at the top of Home:
Your next trip starts in X days

If the trip starts today:
	•	Use copy: Starts today

Rules
	•	Hide entirely if no upcoming trips
	•	No urgency language
	•	No CTA

⸻

	4.	AVAILABILITY MATCH NUDGE (HOME)

Purpose
Surface alignment opportunities without prompting conversation.

Trigger
If:
	•	User has open availability
	•	And a friend has overlapping availability or a trip

Display
Show one lightweight nudge on Home.

Example structure (copy can vary slightly):
You and 2 friends are free Jan 15–17

Rules
	•	Show only one nudge at a time
	•	Nudge must be dismissible
	•	Once dismissed, never show again for that same date range
	•	Dismissal must be stored in the database, scoped to the user and date range
	•	No notifications
	•	No urgency language

Tap behavior
	•	Route directly to the most relevant friend’s profile
	•	Do not route to filtered lists or generic screens

⸻

	5.	FRIEND ORDERING BY RELEVANCE (INVISIBLE)

Purpose
Make the Friends list feel naturally useful without explanation.

Sorting signals to include
	•	Upcoming trip overlap
	•	Pass compatibility
	•	Shared availability

Explicitly exclude
	•	Terrain overlap (do not include this yet)

Rules
	•	No visible scores
	•	No labels
	•	No “sorted by relevance” text
	•	Sorting should feel natural and unexplained

⸻

	6.	RIDER-AWARE COPY (GLOBAL)

Purpose
Increase personalization through precise language.

Behavior
Adjust microcopy based on rider type:
	•	Your skis versus Your board
	•	Add ski day versus Add ride day

Rules
	•	If rider type is not set, default to neutral copy such as “Your gear”
	•	No slang
	•	No tone changes beyond specificity

⸻

	7.	SEASONAL AWARENESS (COPY ONLY)

Purpose
Make the app feel aware of the time of year through subtle copy changes.

Defined seasons
	•	Pre-season: October 1 to November 30
	•	Mid-season: December 1 to March 15
	•	Spring: March 16 to April 30

Outside these ranges, use neutral copy.

Scope
	•	Empty states
	•	Helper text
	•	Small prompts

Rules
	•	Copy changes only
	•	No layout changes
	•	No banners
	•	No notifications

⸻

GLOBAL GUARDRAILS
	•	No feeds
	•	No comments
	•	No likes or reactions
	•	No new navigation
	•	No gamification
	•	All personalization must be calm, optional, and reversible

⸻

ACCEPTANCE CRITERIA
	•	Personalization reduces friction without adding UI clutter
	•	Home feels alive but calm
	•	Friends list feels naturally relevant
	•	Defaults feel tasteful, not automated
	•	App remains mobile-first and scannable
