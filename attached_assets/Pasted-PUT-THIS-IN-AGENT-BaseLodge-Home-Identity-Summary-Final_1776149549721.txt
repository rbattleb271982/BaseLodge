PART 2 — FIX IDENTITY LINE FORMATTING

Current output is wrong because it is dumping raw combined values.

Correct the identity line so it renders a clean human-readable summary.

FINAL FORMAT RULES

Rider types:
- If one rider type:
  - Skier
  - Snowboarder
- If two rider types:
  - Skier + Snowboarder

Use the plus sign ONLY for combining rider types.

Everything else in the identity line should be separated by centered dots:

Examples:
- Skier · Intermediate · Ikon
- Snowboarder · Advanced · Epic
- Skier + Snowboarder · Intermediate · Ikon
- Skier + Snowboarder · Intermediate · Epic · Ikon

Important:
- Do NOT use raw commas like:
  Skier,Snowboarder
- Do NOT use "+" for pass names
- Do NOT introduce double separators
- Omit missing parts cleanly

Use the app’s existing clean display logic where possible instead of dumping raw stored values.

Identity line should remain:
- muted
- sans-serif
- normal case
- directly under the greeting

PART 3 — EQUIPMENT BLOCK LAYOUT (OPTION C DIRECTION)

Implement the equipment area as aligned rows with a fixed label column.

No “Setup” label.

Examples:

Skier full:

BOOTS    Salomon S/Pro

Snowboarder full:

- keep the rows clean and editorial, not form-like
- no cards, no input boxes, no heavy borders

Use aligned rows, not a two-column grid with stacked label/value cells.

PART 4 — EQUIPMENT DATA LOGIC

Use the user’s active equipment data.

Display rules:

If rider type includes skier:
- use ski brand + model for SKIS row
- use boot brand + model for BOOTS row if available

If rider type includes snowboarder:
- use snowboard brand + model for BOARD row
- use boot brand + model for BOOTS row if available

If both skier and snowboarder are selected:
- show the equipment that matches the user’s active/primary setup logic already used by the equipment model or route context
- do not dump both entire disciplines into Home unless the existing active equipment object explicitly supports that cleanly

Fallback rules:
- If only skis/board exists, show that row only
- If only boots exist, show BOOTS row only
- If no valid equipment exists, show:
  Add your setup

Formatting rules:
- Prefer brand + model
- If both exist → show "Brand Model"
- If only brand exists → show brand
- If only model exists → show model
- If neither exists → treat as missing

Do NOT show:
- bindings
- poles
- helmets
- other secondary gear
