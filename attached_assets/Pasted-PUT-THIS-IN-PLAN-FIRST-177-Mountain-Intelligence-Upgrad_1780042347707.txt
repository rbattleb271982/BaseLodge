#177 - Mountain Intelligence Upgrade v1

What & Why

Resort Intelligence tells us what users have already done.

Mountain Intelligence should tell us where future demand exists.

Goal:

Answer:

* Which resorts attract attention?
* Which resorts convert interest into action?
* Which resorts have latent demand?
* Which resorts are over-performing?
* Which resorts are under-performing?
* Which destinations could become partnership opportunities?

Use existing production data only.

Do not add:

* New tracking vendors
* New databases
* New event systems

Inspect and reuse the existing /admin/resort-intelligence implementation where possible.

Do not duplicate logic unnecessarily.

Before implementation, perform a feasibility audit.

---

## PHASE 1 — FEASIBILITY AUDIT

Audit all available mountain-level signals.

At minimum inspect:

1. Resort Detail Views
2. Wishlist Adds
3. Planned Trips
4. Group Trips
5. Friend Activity
6. Availability Overlaps
7. Pass Compatibility
8. Invite Activity
9. Resort Page Traffic

For each signal classify:

GREEN
Directly measurable today

YELLOW
Approximation available

RED
Not reliably measurable

Provide:

* Source model
* Query approach
* Current counts
* Data quality concerns
* Recommendation

Return the audit as a table with:

* Signal
* Classification
* Source Model
* Current Count
* Confidence
* Data Quality Concerns
* Recommendation

Do not build until audit is complete.
