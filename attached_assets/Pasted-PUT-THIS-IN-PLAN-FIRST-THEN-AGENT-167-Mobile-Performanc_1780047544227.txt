#167 - Mobile Performance Audit & Optimization

What & Why

The biggest remaining issue affecting the perceived quality of BaseLodge is screen-to-screen responsiveness.

The app looks good and the Home screen content feels appropriate. The next priority is reducing the feeling that users are navigating a website inside a wrapper.

Focus on real-world perceived performance for iPhone 15 Pro.

No new features.
No redesigns.
No model changes.
No migrations.

Done looks like

- Home feels faster when opening
- Friends feels faster when opening
- Mountains feels faster when opening
- Profile feels faster when opening
- Trip Detail feels faster when opening
- No unnecessary loading flashes
- No duplicate queries
- No redundant API calls
- No unnecessary template work
- Navigation between major tabs feels noticeably more responsive
- Existing functionality remains unchanged

Out of scope

- New UI
- New analytics
- New admin pages
- New features
- Messaging work
- Availability redesign
- PostHog work

Audit Areas

1. Home

Measure:
- total route time
- DB query count
- slowest queries
- ideas_engine execution time
- friend trip feed generation time
- overlap generation time

Confirm:
- no duplicate queries
- no unnecessary relationship loading
- no repeated calculations already available in memory

2. Friends

Measure:
- route execution time
- query count
- friend lookup cost
- trip overlap generation cost
- wishlist overlap cost

Confirm:
- no N+1 query patterns
- no repeated user lookups

3. Mountains

Measure:
- route execution time
- mountain card generation
- filtering cost

Confirm:
- no unnecessary joins
- no repeated calculations

4. Profile

Measure:
- route execution time
- equipment loading
- stats loading
- mountain history loading

Confirm:
- no duplicate fetches

5. Trip Detail

Measure:
- route execution time
- participant loading
- invite loading
- overlap calculations

Confirm:
- no duplicated work

- scripts that can load after first paint
