Investigate and fix the brief black-screen flash some users see when switching between tabs/pages in BaseLodge on mobile devices.

Context:
- One tester has dark mode enabled on their phone.
- When switching tabs (Home, Trips, Friends, Mountains), they briefly see a black flash between page transitions.
- On another device in light mode, the same transition appears as a slight beige lag instead of black.
- This strongly suggests the WebView/root background is temporarily rendering dark before the next page paints.

Goal:
Ensure BaseLodge consistently shows the cream/beige app background during all route transitions, regardless of device dark mode.

Requirements:

1. Investigate the full page/background stack:
- html
- body
- app shell containers
- base templates
- safe-area wrappers
- any Capacitor/webview-related styles
- any dark-mode media queries
- any transparent backgrounds

2. Determine:
- Which layer is rendering black during transitions
- Whether the issue is purely CSS/webview repaint related
- Whether any native Capacitor background configuration contributes

3. Implement the safest web-first fix:
- Ensure the root app background is consistently the BaseLodge cream/beige color during transitions
- Prevent dark mode from temporarily forcing black backgrounds
- Preserve the current BaseLodge visual design
- Avoid introducing white flashes instead

4. Investigate whether:
color-scheme
prefers-color-scheme
or missing root backgrounds are contributing.

5. If a native Capacitor-level improvement is recommended:
- Explain it separately
- Do NOT implement native changes yet

6. Do not:
- redesign navigation
- convert routing architecture
- modify native iOS or Android files
- change dark/light theme support globally unless required for the transition fix

7. Use checkpoints sparingly.

Testing:
- Test rapid tab switching between Home, Trips, Friends, Mountains
- Test simulated dark mode conditions if possible
- Confirm no black flash appears during transitions
- Confirm no new white flash/regression appears
- Confirm the cream/beige background remains visually consistent

Do not finish until fully tested in Replit and summarize:
- root cause
- files changed
- whether native follow-up work is recommended
