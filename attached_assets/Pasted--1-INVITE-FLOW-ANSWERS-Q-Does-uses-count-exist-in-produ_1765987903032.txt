🔴 1. INVITE FLOW — ANSWERS
Q: Does uses_count exist in production, or was it from an older version?
➡️ uses_count does NOT exist in production.
It was referenced from an older/abandoned multi-use design and should be treated as fully deprecated.
Production schema is confirmed as:
(id, token, inviter_id, created_at, used_at)
No code should reference uses_count or max_uses anywhere.
Enforcement is single-use via used_at only.
Q: Should the Invite tab show anything after generating a token?
➡️ Yes, but minimal.
The Invite tab should:
Allow generating an invite
Show the shareable link and/or QR code
Allow copying or sharing
It should NOT:
Show invite history
Show usage counts
Show past invites list
Keep it lightweight and functional.

🟤 5. EQUIPMENT (Gear → Equipment) — ANSWERS
Q: Inline edit — accordion or replace text?
➡️ Replace display text with inputs in-place.
No accordion animation needed
No modal
Simple inline swap:
Display mode → Edit mode
Edit link → Save button
Save → back to display mode
Q: Units — fixed or selectable?
➡️ Fixed units only. No user selection.
Length = centimeters (cm)
Width = millimeters (mm)
Do not add unit toggles
Do not store units separately
Q: What are the Equipment fields?
➡️ Fields are:
For each (Primary / Secondary):
Discipline (Skier / Snowboarder) – if already captured elsewhere, reuse
Brand
Model (if present today)
Length (cm)
Width (mm)
Display only fields that are present.
