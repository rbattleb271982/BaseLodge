#177 - Admin Dashboard — Active Today Drilldown

What & Why

The Founder Pulse "Active" KPI shows a count but not who. Add a tappable drilldown: tapping the Active card opens a bottom sheet listing active-today users (full name + state + last active time). Tapping any user row navigates to an admin-only user detail page.

This is an admin-only feature designed to make the dashboard operational without changing any existing metrics, schemas, or user-facing functionality.

Done Looks Like

- Active card (desktop strip and mobile card) is tappable
- Cursor pointer and subtle hover/active state
- Tapping opens a native-style bottom sheet
- Bottom sheet includes:
  - Rounded top corners
  - Drag handle
  - Close X button
  - Title
  - Subtitle
  - Scrollable list of users
- User list shows:
  - Full name
  - State abbreviation inline with name
  - Last active time on the right
- Empty state renders correctly when count is 0
- Tapping a user row opens the admin user detail page
- User detail page loads without error
- Mobile-friendly
- No horizontal scrolling
- No dashboard metric changes
- No dashboard layout changes
- No user-facing changes
- No console errors
- No broken routes
- The number displayed in the Active KPI exactly matches the number of users returned by the drilldown list

Out of Scope

- Drilldowns on Sessions, New Users, Trips, WAU, MAU, etc.
- Editing users
- Messaging users
- Deleting users
- User impersonation
- Push notifications
- Exporting data
- User-facing routes
- Schema changes
- Model changes

Implementation

1. Active Today API Endpoint

Add an admin-only endpoint:

GET /admin/api/active-today-users

Requirements:

- Admin authentication required
- Return JSON array:

[
  {
    "id": 123,
    "name": "Richard Battle-Baxter",
    "state": "CO",
    "last_active_at_iso": "2026-05-29T15:22:00"
  }
]

IMPORTANT:

Use the exact same definition of "active today" currently used by the Founder Pulse Active metric.

Do not introduce a new definition of active users.

The Active KPI count and drilldown list must always match.

Order results by most recent activity descending.

Name format:

"First Last"

(last name optional if missing)

State:

User.home_state abbreviation

Empty string if unavailable.
