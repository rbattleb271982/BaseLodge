#178 - Admin Dashboard — Active Today Drilldown

What & Why

The Founder Pulse "Active" KPI shows a count but not who. Add a tappable drilldown: tapping the Active card opens a bottom sheet listing active-today users (full name + state + last active time). Tapping any user row navigates to a new admin-only user detail page at /admin/users/<user_id>.

Admin-only. No schema changes, no model changes, no user-facing changes, and no changes to existing metrics or the definition of "active today."

Done Looks Like

- Active card (desktop strip item + mobile card) is tappable
- Cursor pointer and subtle hover/active feedback
- Tapping opens a native-style bottom sheet
- Rounded top corners (18px+)
- Drag handle
- Close × button
- Smooth slide-up animation
- Sheet title = "N Active Today"
- Sheet subtitle = "Users who have been active in the app today"
- User rows display:
  - First Last (ST)
  - Relative last active time
- No avatars
- No profile photos
- No email addresses
- No user IDs
- Rows are tappable
- Row click opens /admin/users/<user_id>
- Empty state renders correctly
- Sheet closes on:
  - X click
  - Overlay click
  - Escape key
- /admin/users/<user_id> returns 200 for valid users
- /admin/users/<user_id> returns 404 for invalid users
- User detail page contains:
  - Account section
  - BaseLodge Activity stat cards
  - Recent Activity
  - Mountains
  - Trips
- Graceful "Not available" fallbacks
- Active KPI count exactly matches drilldown row count
- No horizontal scrolling on mobile
- Existing dashboard metrics unchanged
- Existing dashboard layout unchanged
- Existing dashboard styling unchanged
- No console errors
- No broken routes
- No user-facing changes

Out of Scope

- Drilldowns for any other KPI
- Editing users
- Messaging users
- Deleting users
- User impersonation
- Push notifications
- Exporting data
- Schema changes
- Model changes
- User-facing routes or templates

Implementation

1. Active Today API Endpoint

Add:

GET /admin/api/active-today-users

Requirements:

- Admin-auth required
- Use the exact same predicate already used by Founder Pulse Active:

User.last_active_at >= _admin_today_start_utc()

- Do not introduce a new definition of active users
- Order by User.last_active_at DESC

Return:

[
  {
    "id": 123,
    "name": "Richard Battle-Baxter",
    "state": "CO",
    "last_active_at_iso": "2026-05-29T15:22:00"
  }
]

Name rules:

- "First Last"
- Omit last name if null
- If both names are missing, render "Unknown User"

State rules:

- home_state if available
- otherwise ""

6. Admin User Detail Route

Add:

GET /admin/users/<int:user_id>

Admin-auth required.

404 if user not found.

Use existing models only.

Query:

User record

Trips created:
SkiTrip.query.filter_by(user_id=uid)
.order_by(SkiTrip.start_date.desc())
.limit(5)

Trips joined count:
SkiTripParticipant
where user_id=uid
and status='accepted'

Friend count:
Friend accepted relationships

Days active:
Distinct activity dates from Activity

Recent activity:
Latest Activity rows
+
Latest MountainPageView rows

Merge
Sort descending
Limit 20

Map to human-readable labels.

Active today:

user.last_active_at >= _admin_today_start_utc()

Render:

templates/admin_user_detail.html
