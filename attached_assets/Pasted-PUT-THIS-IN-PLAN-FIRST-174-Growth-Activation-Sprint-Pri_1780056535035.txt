#174 - Growth & Activation Sprint Prioritization

What & Why

The Admin Intelligence initiative is complete for now.

The Executive Dashboard now surfaces the three biggest opportunities:

1. Push Reliability
2. Availability Adoption
3. User Growth

Goal:

Determine which problem to solve first and what the next sprint should include.

This is a prioritization audit, not a dashboard build.

Do not create:

* New dashboards
* New admin pages
* New models
* New migrations
* New vendors
* New tracking systems

Use existing production data only.

---

## PHASE 1 — PUSH RELIABILITY AUDIT

Current signal:

* Push failure rate is approximately 30%
* 39 of 132 push attempts failed

Audit:

1. Which push events are failing?
2. Which users have repeated failures?
3. Are failures concentrated by platform?

   * iOS
   * Android
   * Web / unknown
4. Are failures tied to stale or inactive tokens?
5. Are failures tied to missing tokens?
6. Are failures tied to specific event types?
7. Can invalid tokens be safely deactivated?
8. What data is available in MessageEventLog error fields?
9. What can be fixed without OneSignal webhooks?
10. What requires OneSignal webhook/API integration later?

Classify possible fixes:

GREEN
Can be implemented now with existing data/code

YELLOW
Can be partially improved now

RED
Requires new integration or external dependency

Deliver:

* Root cause hypothesis
* Highest-impact fix
* Estimated effort
* Risk
* Recommended implementation plan

## PHASE 2 — AVAILABILITY ADOPTION AUDIT

Current signal:

* 20 users have a pass
* 6 users have availability
* 18 users have a pass but no availability set

Audit:

1. Where is availability currently surfaced?
2. Is availability part of onboarding?
3. Is availability easy to find on Home?
4. Is availability easy to find on Profile / Settings?
5. Are there existing nudges for users with pass but no availability?
6. Are there existing CRM audience definitions for this group?
7. Can this be improved with UI only?
8. Can this be improved with messaging only?
9. Can this be improved with both?

Classify possible fixes:

GREEN
Can be implemented now

YELLOW
Needs light product/content changes

RED
Requires larger redesign or new system

Potential fixes to evaluate:

* Home screen nudge for users with pass but no availability
* Profile/settings prompt
* Post-onboarding prompt
* CRM audience export/list
* Push/email nudge later
* Copy changes to explain why availability matters

Deliver:

* Highest-impact availability adoption fix
* Recommended user experience
* Estimated effort
* Risk
* Implementation plan
