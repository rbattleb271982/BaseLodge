# SERVER IMPLEMENTATION (app.py)

## AUTH ROUTE (/auth POST)

### auth_error
Fire for server-side validation/auth failures only:
- missing_fields
- password_too_short
- email_taken
- invalid_credentials

Example:
ph_analytics.track(None, 'auth_error', {
  'error_type': 'missing_fields'
})

Do NOT fire for client-side UI issues.

---

### signup_completed (email signup)

After:
- user is created
- db.session.commit() succeeds
- login_user(new_user) is called

Fire:
ph_analytics.track(new_user.id, 'signup_completed', {
  'method': 'email',
  'signup_source': 'invite' OR 'organic'
})

Determine signup_source from session/invite logic if available.

---

### login_completed (email login)

After:
login_user(user)

Fire:
ph_analytics.track(user.id, 'login_completed', {
  'method': 'email'
})

---

## LOGOUT ROUTE

Before logout_user():

1. Capture:
