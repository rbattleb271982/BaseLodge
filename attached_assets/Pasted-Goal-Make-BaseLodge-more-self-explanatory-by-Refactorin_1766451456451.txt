Goal
Make BaseLodge more self-explanatory by:
Refactoring the profile stats row to a clearer 3-column layout with labels above numbers
Adding short, contextual helper sentences across key screens to explain what users are supposed to do and why
This is UX copy + layout only.
Do not change backend logic or data models.
PART 1 — PROFILE STATS ROW (UI REFACTOR)
Layout
One horizontal row
3 equal-width columns
Mobile-first
Center-aligned content
No icons
Column structure (top → bottom)
Label
Number
Labels (EXACT COPY — DO NOT CHANGE)
Trips Created
Visited Mountains
Bucket List Mountains
Data mapping (use existing variables)
Trips Created → number of trips created by the user
Visited Mountains → count of visited mountains
Bucket List Mountains → count of bucket list mountains
Interaction
Entire column is tappable
No “Edit” text
No tooltips
Routing
Trips Created → Trips tab/page
Visited Mountains → Visited Mountains view/edit
Bucket List Mountains → Bucket List view/edit
Typography
Label: ~12–13px, regular/medium, slightly muted
Number: ~26–32px, semibold/bold, primary color
Small vertical spacing between label and number
Explicitly do NOT
❌ Add icons
❌ Stack into rows
❌ Rename labels
❌ Change backend logic
PART 2 — MICROCOPY / HELPER TEXT (GUIDANCE)
Add short, single-sentence helper copy in the following locations.
Do not add modals or tooltips — this should feel calm and lightweight.
1️⃣ Trips Tab (Top of Screen)
Add helper text above the trip list:
“Add trips you’re planning so friends can see where you’re skiing.”
2️⃣ Trips Tab — Empty State (0 Trips)
Headline
No trips yet
Subtext
Add a trip and your friends will see where you’re going.
CTA
Add a trip
3️⃣ Profile Screen (Top, Under Name / Identity)
Add one sentence below the profile header:
“Your profile helps friends understand how you ski.”
4️⃣ Profile — Equipment Section
Above equipment fields, add helper copy:
“Optional — add what you ride so friends get a better sense of your setup.”
If there is secondary helper text space, include:
“Not sure? You can leave this blank.”
5️⃣ Profile — Riding Style Field
Add helper copy near the field:
“This helps friends understand what kind of days you enjoy.”
6️⃣ Invite Tab (Top of Screen)
Add helper copy at the top:
“Invite friends so you can see each other’s trips and plans.”
7️⃣ Invite Tab — Empty State
Headline
Build your ski crew
Subtext
Invite friends to share trips and plan around the same dates.
8️⃣ Visited Mountains Screen
Add helper text at top:
“These are mountains you’ve already skied.”
9️⃣ Bucket List Mountains Screen
Add helper text at top:
“Mountains you want to ski someday.”
🔟 Add Trip Screen (Top of Form)
Add helper text above inputs:
“Trips you add here are visible to your friends.”
IMPLEMENTATION RULES (IMPORTANT)
Helper copy = one sentence max
Do not clutter screens
Do not add new fields
Do not introduce onboarding or tutorials
Use existing layout patterns and styles
Tone must feel friendly, calm, and non-instructional
DONE CRITERIA
Users landing on Trips know what to do next
Empty states explain themselves
Profile fields feel optional, not mandatory
No ambiguity about why information is being entered
UI feels clearer without feeling busier
