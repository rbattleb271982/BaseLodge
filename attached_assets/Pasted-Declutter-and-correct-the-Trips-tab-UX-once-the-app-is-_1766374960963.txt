Declutter and correct the Trips tab UX once the app is fully populated, so it is scannable, intention-aligned, and consistent with user expectations.
1️⃣ Remove “Next trip” from the Trips header
Change
Remove the “Next trip · Jan X–Jan Y” text from the top profile card on the Trips tab.
Reason
This duplicates information already visible in the trip list.
It adds noise once users have multiple trips.
Acceptance criteria
No “Next trip” copy appears anywhere on the Trips page header.
The trip list itself is the only source of upcoming trip information.
2️⃣ Availability banner behavior (“You and X friends are free…”)
Current problem
The banner claims a specific free date even when availability is not open.
It only references a single date instead of all overlapping dates.
New behavior (LOCKED)
Display rules
Show the availability banner only if there is at least one overlapping free date with friends.
Do not reference a specific date unless availability is actually open.
Copy update
Replace:
“You and 17 friends are free Jan 20”
With:
“You and 17 friends have overlapping availability”
CTA remains:
“See availability →”
Tap behavior
When the user taps “See availability →”:
Navigate to the Open Avail. tab
Automatically scroll/focus to:
All dates where the user overlaps with ≥1 friend
Visually highlight all overlapping dates
If availability is NOT open:
Still navigate to Open Avail.
Show the availability toggle and explanation
Do not claim any specific free dates
Acceptance criteria
The banner never promises a date unless availability is open.
Multiple overlapping dates are supported and visible.
Banner reflects reality at all times.
3️⃣ Trip Overlaps tab — prevent unintended editing
Current problem
Tapping a date in Trip Overlaps opens the Edit Trip screen.
Correct behavior
Trip Overlaps is read-only discovery, not editing.
Change
Remove any navigation from Trip Overlaps dates to Edit Trip.
Tapping a date should:
Show overlapping friends
Expand inline details
Or open a read-only trip view
Editing is only accessible from My Trips.
Acceptance criteria
No interaction in Trip Overlaps leads to an Edit Trip screen.
4️⃣ My Trips card — inline, single-line summary
New card format (LOCKED)
Each trip in My Trips should render as a single inline row:
Jan 17–20 · Alta, UT · Ikon
Information rules
Show on the card:
Dates
Mountain + state
Pass
Do NOT show on the card:
Number of nights (e.g. “3+ nights”)
Equipment (“Have own equipment”)
Planning metadata
Social counts
Detail view behavior
Tapping the card opens the Trip Detail view
Trip Detail view may show:
Number of nights
Equipment
Social info
Edit controls (owner only)
Friend visibility
Friends never see edit affordances.
Friends only see read-only trip details.
Acceptance criteria
My Trips list is fully scannable in one glance.
No extra metadata appears in the list view.
5️⃣ General UX constraints
Mobile-first layout
No new copy unless explicitly specified above
No new features added
Changes are limited to Trips-related templates, routes, and client logic
Final acceptance checklist
 “Next trip” header removed
 Availability banner reflects all overlapping dates, not one
 Banner never claims availability when it’s not open
 Trip Overlaps never opens Edit Trip
 My Trips cards are single-line inline summaries
 Nights & equipment only appear in trip detail view
