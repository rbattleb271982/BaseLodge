INVITE & ACCEPT LOGIC
================================
- Invited users must explicitly ACCEPT or DECLINE
- Users are NOT considered “going” until ACCEPTED

- Create TripParticipant with status = ACCEPTED
- If the user already has a solo trip for the same destination and dates:
  - Automatically archive / hide that solo trip
  - Link it via converted_to_trip_id
  - Carry over all personal trip details as-is
    (equipment, lodging, transportation, lessons)
- If the user had no solo trip:
  - Join shared trip with empty personal details

================================
DECLINE LOGIC
================================

- Declined users are not counted as going
- Declined users can be re-invited later
- No permanent declined participant record is required in the shared trip view
