PRODUCT RULES TO IMPLEMENT
====================================

1. V1 idea types
Only these 3 idea types should exist in the unified feed:
- trip_overlap
- availability_overlap
- wishlist_overlap

2. Pass logic
Pass overlap is NOT a trigger.
Pass overlap is:
- a scoring boost
- a subtitle enhancement
- useful context
A shared pass alone should never create a card.

3. Feed philosophy
The Ideas tab is a highlight reel, not an exhaustive data dump.
Show only the best 3–5 total ideas across all types.
All idea types compete in one ranked pool.

4. Product principle
Combine signals into one stronger idea whenever possible.
Do not show weak, fragmented, repetitive signals if they can be merged.

PHASE E — Unified scoring across types

6. Add a unified scoring layer for all IdeaCard candidates.

Use this approved scoring philosophy:
Rank by “how close is this to becoming a real trip?”

Recommended scoring model:

Trip Overlap
- +50 base
- +40 if user's availability overlaps the trip
- +30 if pass alignment exists
- +20 if trip resort matches user's wishlist
- +10 if trip starts within 30 days
- +10 if trip starts within 14 days

Availability Overlap
- +40 base
- +5 per relevant friend, cap at +25
- +25 if a shared wishlist destination meaningfully enriches the card
- +15 if pass overlap meaningfully enriches the card
- +10 if date window starts within 14 days
- +5 if date window starts within 30 days

Wishlist Overlap
- +30 base
- +5 per relevant friend, cap at +25
- +10 if pass overlap exists
- +10 if those same friends also share a meaningful date overlap soon
- +10 if near-term planning window exists, if grounded

Important:
- Do not let pass-only ideas exist
- Use pass as a moderate boost, not dominance
- If a friend trip exists with zero meaningful signal, do not create a card just because the friend has a trip

PHASE F — Diversity selection

7. After generating and scoring all candidate cards, enforce diversity before final selection.

Target:
Return only the best 3–5 total cards.

Selection method:
- Start with candidates sorted by score descending
- Select the top card
- For remaining candidates, apply soft penalties before choosing the next card:
  - same friend overlap as a selected card → meaningful penalty
  - same date window → meaningful penalty
  - same resort → meaningful penalty
  - same idea type → smaller penalty

Suggested penalty ranges:
- same friend: ~30%
- same resort: ~25%
- same date window: ~20%
- same idea type: ~15%

Use grounded, readable code. It does not need to be mathematically fancy. It just needs to prevent repetitive feeds.

Important:
- Do NOT hard-slot one card per type
- Diversity should be soft, not rigid
- Stronger ideas should still win
