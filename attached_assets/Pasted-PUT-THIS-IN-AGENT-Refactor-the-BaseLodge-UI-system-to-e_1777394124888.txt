Refactor the BaseLodge UI system to enforce strict cross-app cohesion across auth, onboarding, and logged-in screens.

Use the uploaded #61 Cross-App UI System Unification brief as the source of truth, with these clarifications:

1. Primary buttons

* There should be only one dominant primary action per screen or viewport.
* Primary buttons should be burgundy, white text, 52px tall, 12px radius, 15px/600.
* Secondary actions must be visually demoted as outlined, text, or muted actions.

2. Navigation indicators

* Use chevron › only for list rows that navigate to a new screen.
* Use arrow → only for inline text actions like Invite, Edit, Add availability.
* Do not mix arrows and chevrons within the same interaction type.

3. Inputs

* All single-line inputs should be white-filled, 1px warm gray border, 12px radius, 52px min-height.
* Input text should use var(--bl-color-text).
* Placeholder text should use var(--bl-color-text-tertiary).

4. Pills and segmented controls

* Pills are only for selection inputs like rider type, terrain, pass, and equipment selection.
* Pills should use 12px radius, not full pill shape.
* Segmented controls such as Log in / Create account may keep pill radius because they are a distinct control type.
