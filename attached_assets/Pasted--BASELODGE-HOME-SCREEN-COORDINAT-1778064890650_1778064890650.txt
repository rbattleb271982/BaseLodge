║ BASELODGE — HOME SCREEN COORDINATION LAYER RESTRUCTURE      ║
╚══════════════════════════════════════════════════════════════╝

Goal:
Begin restructuring Home into the new BaseLodge coordination layer.

Home should no longer be centered around “Your next trip.” Trips owns committed plans. Home should focus on:
- identity
- smart action pills
- friends-only Happening
- ranked coordination opportunities

IMPORTANT:
- Do NOT remove the Trips tab.
- Do NOT remove existing trip functionality.
- Do NOT remove the Ideas route yet.
- Do NOT build the Mountains page beyond the existing placeholder.
- Do NOT add snow, weather, conditions, powder, forecasts, or resort operations logic.
- Do NOT add friend avatars/profile bubbles.
- Keep this mobile-first and visually restrained.
- Only use checkpoints if absolutely necessary.

────────────────────────────────────
NEW HOME ORDER
────────────────────────────────────

Home should display in this order:

1. Welcome card
2. Smart action pills
3. Happening
4. Opportunities

“Your next trip” should no longer be the dominant Home module. If existing next-trip logic must remain temporarily for safety, demote it or do not render it as the main centerpiece.

────────────────────────────────────
SECTION 1 — WELCOME CARD
────────────────────────────────────

Keep existing welcome card structure:
- Welcome, Richard.
- identity line
- equipment line
- stats
- profile initials bubble inside card

Do not materially redesign this section.

────────────────────────────────────
SECTION 2 — SMART ACTION PILLS
────────────────────────────────────

Add/adjust smart pills below the welcome card.

Pills:
- Ideas {count}
- Requests {count}
- Add dates

Rules:
- No “All” pill.
- Requests pill only appears if request count > 0.
- Add dates only appears if user has no availability or stale/missing availability.
- Ideas count should reflect actionable coordination opportunities only.
- Pills do not navigate away from Home.
- Pills should act as inline focus filters.

Style:
- compact pill shape
- typography-first
- maroon active pill if selected
- quiet outline/cream pills otherwise
- no dashboard tiles
- no chunky cards

────────────────────────────────────
SECTION 3 — HAPPENING
────────────────────────────────────

Move Happening above Opportunities.

Happening should be friends-only ambient activity.

Examples:
- Sam planned Vail
- Nina added Alta
- 3 friends skiing this weekend

Rules:
- No avatars.
- No profile bubbles.
- No reactions/comments/likes.
- No weather/snow/conditions.
- Keep it quiet and lightweight.
- Show the section if there is at least one meaningful friend activity signal.

If existing happening_signals logic already exists but is not rendered, wire it safely into Home.

────────────────────────────────────
SECTION 4 — OPPORTUNITIES STREAM
────────────────────────────────────

Create a unified Opportunities section.

Header:
- Left: Opportunities
- Right: “5 today” when there are 5 visible opportunities

Default:
- show up to 5 ranked opportunities

Rules:
- unified stream only
- no grouped sub-sections
- no giant hero card
- no photography-heavy cards
- no friend profile bubbles
- no infinite feed behavior

Opportunity types that count:
1. Friend trip alignment
   Example: 3 friends planning Aspen

2. Availability overlap
   Example: Your dates overlap with Nina

3. Wishlist coordination overlap
   Example: You and Nina saved Snowbird

Wishlist overlaps should count as Ideas when they have coordination potential.

Each opportunity row/card should:
- be separate, not inside one giant grouped card
- be lightweight and tappable
- open the relevant mountain page
- have a subtle dismiss control
- be score-ranked
- feel calm and premium

Dismissal:
- dismissed opportunities should not keep showing.
- use existing dismissed insight/card pattern if available.
- keep dismissal subtle.

────────────────────────────────────
PILL BEHAVIOR
────────────────────────────────────

Tapping Ideas:
- stays on Home
- expands/re-focuses the Opportunities section
- shows additional ranked coordination opportunities if available
- does not navigate to /ideas

Tapping Requests:
- stays on Home
- focuses request/connect request content
- hides or deemphasizes unrelated opportunity content

Tapping Add dates:
- stays on Home
- opens or reveals availability setup inline if feasible
- if inline setup is too large for this phase, route to the existing availability page as a temporary fallback, but note it clearly
