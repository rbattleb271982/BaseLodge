PART 1 — CREATE MESSAGEEVENTLOG MODEL
--------------------------------------------------

Create a new SQLAlchemy model:

MessageEventLog

Fields:

id → Integer primary key

event_name → String(120), indexed, required
event_version → Integer, default=1, required
category → String(50), indexed, required

actor_user_id → FK nullable
recipient_user_id → FK nullable

object_type → String(80), nullable
object_id → Integer nullable

channel → String(40), nullable

delivery_status → String(40), indexed, required
Allowed:
- pending
- sent
- skipped
- failed

suppression_reason → String(80), nullable

provider → String(40), nullable
Allowed:
- onesignal
- sendgrid
- apns
- fcm
- internal

provider_message_id → String(255), nullable

payload_json → JSON required

message_title → String(255), nullable
message_body → Text, nullable

error_message → Text, nullable

retry_count → Integer default=0 required

created_at → UTC datetime default now
processed_at → UTC datetime nullable
sent_at → UTC datetime nullable

Relationships:
- actor_user relationship
- recipient_user relationship

Use safe lazy loading patterns consistent with the existing codebase.
