==============================
WISHLIST SECTION LOGIC
==============================

There are THREE mutually exclusive states.
Only ONE may render at a time.

STATE A — User has wishlist mountains AND overlaps exist
- Show:
  "Wishlist · N matches" card
- Behavior:
  Opens Wishlist modal (existing behavior)

STATE B — User has wishlist mountains BUT NO overlaps
- Show:
  NOTHING
- No header
- No message
- No CTA

STATE C — User has NO wishlist mountains at all
- Show a lightweight CTA row:

Header:
  "Wishlist"

Subtext / Action:
  "Add mountains you want to ski to find matches →"

Action:
- Navigate to the user's profile wishlist editor
  (use existing profile navigation)
