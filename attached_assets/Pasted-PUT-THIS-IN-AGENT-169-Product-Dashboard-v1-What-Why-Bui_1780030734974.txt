#169 - Product Dashboard v1

What & Why

Build a new admin page at /admin/product focused on feature adoption, user behavior, and product usage patterns.

Growth tells us if users arrive.

Activation tells us if they reach value.

Retention tells us if they come back.

Product tells us what they actually use.

Goal:

Answer:

- Which features are being used?
- Which features are being ignored?
- What actions correlate with activation?
- What actions correlate with retention?
- Where should we invest product effort?

Use existing production data only.

Do not add:

- New database tables
- New tracking vendors
- New event systems

Before implementation, perform a feasibility audit and report findings.

--------------------------------------------------
PHASE 1 — FEASIBILITY AUDIT
--------------------------------------------------

Do not build yet.

Audit the following product signals:

1. Pass Added
2. Availability Added
3. Wishlist Usage
4. Trip Creation
5. Group Trip Creation
6. Friend Connections
7. Equipment Setup Completion
8. Profile Completion
9. Resort Detail Views
10. Home Feed Usage
11. Invite Usage
12. Notification Usage

For each metric classify:

GREEN
Directly measurable from production data with no assumptions.

YELLOW
Measurable using a reasonable proxy or approximation.

RED
Cannot be measured reliably from current production data.

Do not classify a metric as GREEN unless the underlying data is already persisted and queryable.

Provide:

- Classification (GREEN / YELLOW / RED)
- Source model(s)
- Query approach
- Data quality concerns
- Current counts
- Recommendation (use / modify / exclude)
