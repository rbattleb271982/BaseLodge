What & Why
Two related improvements to the invite system:
1) Logged-in invite link
Currently, if an authenticated user visits /invite/<token>, the token is stored in session but no friend connection is made. The connection only fires at login/signup, which never happens for an already-logged-in user. This silently drops a meaningful share of invite completions.
2) Auth page signup default
When a user arrives at /auth carrying an invite token in session, the page defaults to the Login tab. Invite recipients are usually new users who need to Sign Up, so the default should match likely intent.
Done looks like
Logged-in user opens a valid invite link → mutual Friend rows are created immediately → redirected to /friends with a success flash message
No duplicate friendships are created
Behavior is idempotent on refresh/reopen
invited_by_user_id is set appropriately
Token is marked used_at
Logged-out user opens invite link → unchanged behavior:
invite_landing.html shown
