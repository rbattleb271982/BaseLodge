You are working on the BaseLodge Flask web app. Implement the following feature end-to-end, testing all flows before finishing. Use checkpoints sparingly.

Goal
Add explicit trip duration to personal ski trips so users can easily discover who is going up for a day trip, filter trips by duration, prevent duplicate overlapping trips, and support future notifications. This is a social coordination feature, not just metadata.

Scope
This feature applies to the SkiTrip model only. Do not apply duration logic to GroupTrip in this iteration.

Data Model Changes
Update the SkiTrip model to include a new required field called trip_duration. Use an enum or string type consistent with the existing codebase.

Allowed values (exact):
DAY_TRIP
ONE_NIGHT
TWO_NIGHTS
THREE_PLUS_NIGHTS

Display labels:
Day Trip
1 Night
2 Nights
3+ Nights

Existing Trips Backfill
There are existing trips in the database. Backfill trip_duration automatically based on current start_date and end_date using the following rules:

Same start and end date → DAY_TRIP
1 night → ONE_NIGHT
2 nights → TWO_NIGHTS
3 or more nights → THREE_PLUS_NIGHTS

Do not make the field nullable.

Trip Creation and Edit Behavior

Create Trip
Require users to explicitly select a duration.
Do not infer duration from dates.
Block saving if duration is missing.

Edit Trip
If trip dates change, auto-update trip_duration based on night count using the same rules as backfill.
Do not show a confirmation modal.
Update silently.

Active Trip Definition
A trip is considered active if end_date is greater than or equal to today.
Active trips include upcoming and in-progress trips.
Past trips (end_date < today) must be ignored for overlap validation, social counts, and filters.
Treat all dates as date-only. Ignore timezones entirely.

Overlapping Trip Validation

Rule
Only one active SkiTrip per user per resort per overlapping date range is allowed.

Scope
Overlap checks apply only to active trips.
The rule is enforced per resort.
Multiple trips to different resorts in the same weekend are allowed.

Behavior
If a user attempts to create or edit a trip that overlaps with an existing active trip at the same resort, block the action and show the following alert:

“You already have a trip to {Resort Name} that overlaps these dates. Want to update that trip instead?”

Buttons:
Edit existing trip → redirect in the same tab to the edit page for the overlapping trip
Cancel → remain on the current form

Do not allow duplicate active trips.

Trip Card UI

Wherever SkiTrips appear (home screen, trip lists, friend cards), display duration clearly and compactly.

Examples:
Sat, Jan 13 · Day Trip
Jan 13–14 · 1 Night

Duration must be visible without clicking and should be more prominent than dates in social contexts.

Home Screen Day-Trip Social Signal

Add a subtle home screen card that answers “Who’s doing a day trip?”

Weekend definition:
Friday through Sunday.

Rules:
