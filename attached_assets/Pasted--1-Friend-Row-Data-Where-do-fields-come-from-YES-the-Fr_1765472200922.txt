✅ 1. Friend Row Data — Where do fields come from?
✔ YES — the Friend model is linked to a User
You should query the User object to retrieve the friend's:
rider_type
pass_type
skill_level
Why?
Most friendship tables only store:
user_id
friend_id
(and maybe timestamps)
The actual profile attributes live on the User model, not the Friend model.
So your row should use something like:
friend_user = User.query.get(friend.friend_id)
Then in Jinja:
{{ friend_user.name }} — {{ friend_user.rider_type }} — {{ friend_user.pass_type }} — {{ friend_user.skill_level }}
⚠️ If your friend list query already joins User, you do NOT need to re-query.
Use whatever variable name represents the actual User object.
