Implement an SMS-based invite-to-connect system using native mobile SMS (no address book access, no SMS sending service).
Core requirements
Tapping “Invite via text” opens the native SMS composer with a prefilled message.
Each invite link can be accepted by up to 5 people.
Each user can have 10 total invite accepts max (early-phase limit).
Limits are enforced on acceptance, not on send.
Invites create a mutual connection between users when accepted.
1️⃣ Database changes
Create a new model: InviteToken
Fields:
id
token (string, unique, random)
sender_user_id (FK → User)
max_uses (int, default = 5)
uses_count (int, default = 0)
created_at
expires_at (7 days from creation)
Update User model
Add:
total_invite_accepts (int, default = 0)
max_invite_accepts (int, default = 10)
Run migrations if applicable.
2️⃣ Invite generation logic
Create a route (POST or GET):
/invite/generate
Logic:
If current_user.total_invite_accepts >= current_user.max_invite_accepts
Block and show message:
“You’ve reached the maximum number of invites for now. We’ll open this up more soon.”
Otherwise:
Generate a new InviteToken
Set max_uses = 5
Set expires_at = now + 7 days
Save token
Redirect user to SMS composer using:
sms:?&body=Hey! I’m using BaseLodge to plan ski trips with friends. Join me here 👉 https://baselodge.app/invite/<TOKEN>
3️⃣ Invite landing route
Create:
/invite/<token>
Logic:
Look up token
If token does not exist or is expired:
Show:
“This invite is no longer valid.”
If uses_count >= max_uses:
Show:
“This invite has already been fully used. Ask your friend to send you a new one.”
If user is not logged in:
Show landing page:
“<Sender First Name> wants to connect on BaseLodge”
CTA:
“Create account to connect”
Secondary: “Log in”
After signup/login, continue acceptance flow
If user is logged in, proceed to acceptance
4️⃣ Invite acceptance logic
On acceptance:
Ensure sender and recipient are not already connected
Create bidirectional connection records
Increment:
InviteToken.uses_count += 1
sender.total_invite_accepts += 1
Save all changes atomically
Redirect recipient to:
Friends page or Home
If sender has hit max_invite_accepts, block further accepts with:
“This user has reached their invite limit for now.”
5️⃣ UI copy (must match exactly)
Button label: Invite via text
Helper text near button:
You can invite up to 10 people total for now.
Invite landing headline:
“<First Name> wants to connect on BaseLodge”
Used-up invite message:
“This invite has already been fully used. Ask your friend to send you a new one.”
6️⃣ Constraints
Do NOT access address books
Do NOT send SMS from backend
Do NOT track recipients
All enforcement happens server-side
Mobile-first behavior only
7️⃣ Testing checklist (must complete)
Invite link works when sent to multiple people
Max 5 accepts per token enforced
Max 10 accepts per sender enforced
Expired token handled gracefully
Duplicate connections prevented
Logged-in and logged-out flows both work
Do not finish until all scenarios above are tested and confirmed.
