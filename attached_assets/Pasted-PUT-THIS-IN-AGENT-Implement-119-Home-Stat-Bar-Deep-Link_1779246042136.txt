templates/profile.html around lines 347-357

Goal:
Replace the existing hash-scroll IIFE with a more reliable handler.

Requirements:
- Keep normal /profile navigation unchanged when there is no hash.
- If window.location.hash exists, wait 300ms.
- Inside the timeout, use requestAnimationFrame before measuring layout.
- Find the target element with document.querySelector(window.location.hash).
- If no target exists, do nothing.
- Replace scrollIntoView with explicit window.scrollTo.
- Use target.getBoundingClientRect().top + window.pageYOffset to calculate the absolute target position.
- Subtract a fixed offset so the row lands clearly visible above the fixed nav and safe-area spacing.
- Use an offset of 140px.
- Use smooth behavior.

Suggested replacement:

(function () {
  if (!window.location.hash) return;

  window.setTimeout(function () {
    window.requestAnimationFrame(function () {
      var target = document.querySelector(window.location.hash);
      if (!target) return;

      var offset = 140;
      var targetTop = target.getBoundingClientRect().top + window.pageYOffset;
      var scrollToY = Math.max(targetTop - offset, 0);

      window.scrollTo({
        top: scrollToY,
        behavior: "smooth"
      });
    });
  }, 300);
})();
