BASELODGE — REAL DEVICE PRODUCTION SIMULATION AUDIT

Objective

Validate the actual user experience on physical devices before submitting the next iOS and Android builds.

This audit must simulate real-world user behavior and verify that every invite flow works correctly across:

* iOS
* Android
* Mobile web
* Logged-in users
* Logged-out users
* New users
* Existing users

Do not modify code.

Only test and report.

## PHASE 1 — DEEP LINK VALIDATION

Generate fresh:

* Friend invite links
* Trip invite links

For each link test:

1. iPhone with app installed
2. iPhone without app installed
3. Android with app installed
4. Android without app installed

Verify:

* Does the link open the app?
* Does the link open the browser?
* Does the correct screen load?
* Is invite state preserved?
* Is trip state preserved?
* Are redirects correct?

Document exact behavior.

Document all differences.
