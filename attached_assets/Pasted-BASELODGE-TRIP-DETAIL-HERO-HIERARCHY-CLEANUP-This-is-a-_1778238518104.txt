BASELODGE — TRIP DETAIL HERO HIERARCHY CLEANUP

This is a visual hierarchy cleanup.

Do NOT redesign the full page.

We audited the trip detail hero and confirmed the mountain name is currently shown twice:
- centered top nav title
- large hero mountain title

We want to remove that redundancy and make the dates the primary hero element.

━━━━━━━━━━━━━━━━━━━━
1. KEEP CENTERED NAV TITLE AS MOUNTAIN LINK
━━━━━━━━━━━━━━━━━━━━

Keep the centered top nav title.

Make it the mountain/resort navigation link for all users when a resort slug exists.

Behavior:
- tapping the centered nav title opens the mountain detail page
- preserve the back button
- do not add blue link styling
- do not underline
- keep it subtle/native/editorial

If no resort slug exists:
- render the nav title as plain text

Important:
Do NOT make the nav title open the resort edit sheet.
The nav title should be navigation only, not editing.

━━━━━━━━━━━━━━━━━━━━
2. REMOVE LARGE HERO MOUNTAIN TITLE
━━━━━━━━━━━━━━━━━━━━

Remove the large duplicate hero mountain title entirely.

This removes the repeated:
“La Parva”

The mountain should appear only in the centered nav title.

Do NOT remove the nav title.

━━━━━━━━━━━━━━━━━━━━
3. PROMOTE DATE LINE AS PRIMARY HERO ELEMENT
━━━━━━━━━━━━━━━━━━━━

The dates should become the primary hero content.

Example:
May 26–27, 2026 · 1 night

Increase the visual prominence of the date line:
- larger than current
- stronger hierarchy
- clear and scannable
- still elegant

For trip owners:
- dates remain tappable/editable
- add a subtle trailing chevron or similar native affordance
- no pencil icon
- no blue link
- no underline

For non-owners:
- dates remain static
- no edit affordance

Do not break the existing date-edit bottom sheet.

━━━━━━━━━━━━━━━━━━━━

Remove the “CONFIRMED” eyebrow from the hero.

Reason:
It duplicates the metadata badge “Going,” which uses the same trip_status field.

Keep the metadata badge:
- Planning

The status should live only in the metadata row.
