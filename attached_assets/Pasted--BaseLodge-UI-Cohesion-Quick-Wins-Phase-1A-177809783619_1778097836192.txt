║ BaseLodge UI Cohesion Quick Wins — Phase 1A       ║
╚════════════════════════════════════════════════════╝

Goal:
Apply low-risk UI cohesion fixes from the post-audit roadmap without redesigning layouts or changing navigation.

This is a quick-win normalization pass only.

Do NOT:
- redesign screens
- change information architecture
- change routes
- change database models
- alter the Home layout structure
- introduce new colors
- introduce new animations
- touch native iOS/Android files
- use checkpoints unless necessary

Tasks:

1. Replace “Let’s Go!” with “Continue”
Find onboarding or final setup CTA copy that says:
- “Let’s Go!”
- “Let's Go!”
Replace it with:
- “Continue”

Keep the button style and behavior unchanged.

2. Fix email input autocapitalization
On login/signup/auth email fields:
- force lowercase typing where appropriate
- set autocapitalize="none"
- set autocomplete="email"
- set inputmode="email"
- ensure email display is not auto-uppercased by CSS
- preserve existing validation and submission logic

3. Remove fake-link underlines from non-clickable Home metadata
On the Home welcome card / metadata line, remove underline styling from decorative metadata such as:
- rider type
- skill level
- pass type

Important:
- Keep true links styled as links.
- “Add your gear →” should remain visually link-like because it is tappable.
- Do not remove underline globally from all links.

4. Normalize disabled primary CTA color
Find disabled primary CTA styles that differ across onboarding and plan-a-trip.
Create or reuse one shared disabled state token:
- disabled fill should be a muted/low-opacity burgundy treatment
- disabled text should remain readable
- apply consistently to primary CTA buttons only

Do not change enabled CTA styling.

5. Normalize badge styling for NEW / GOING / PRIMARY SETUP
Create or reuse a shared mini-badge class for:
- NEW
- GOING
- PRIMARY SETUP

Badge behavior:
- uppercase
- compact pill shape
- burgundy fill
- white text
- subtle letter spacing
- consistent padding/radius

Replace obvious one-off badge styling, especially:
- lowercase inline “new”
- plain text “Going” status
- inconsistent PRIMARY SETUP pill styling

Do not change the underlying data logic.

6. Normalize obvious capitalization drift in displayed labels
Only fix visible display labels where the mapping is obvious and safe:
- “epic” → “Epic”
- “Cross-Country” / “Cross-country” should be displayed consistently as “Cross-country”
- “Social (along for the ride)” and “Social / après” should not both appear for the same concept if they are already mapped in templates

Important:
- Do not create a major content-model refactor in this pass.
- If labels are centralized already, update the central source.
- If labels are scattered, make only safe display-level fixes and report where a future centralization pass is needed.

Testing requirements:
- Start the app successfully.
- Confirm Python syntax passes.
- Confirm affected templates render without Jinja errors.
- Test login/signup screens.
- Test onboarding final step.
- Test Home welcome card metadata.
- Test at least one disabled CTA state.
- Test Friends list / Friends’ Trips / Mountain detail / Trip detail areas where NEW, GOING, or PRIMARY SETUP appear.
- Confirm no layout redesign occurred.
- Confirm bottom nav still works.

At the end, report:
- Files changed
- Exact text/style changes made
- Which roadmap items were crossed off
- Any items deferred because they require a larger shared-component refactor
